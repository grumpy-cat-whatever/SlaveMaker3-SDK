Core Buttons
Slave Maker 3 SDK Version 8

Introduction
There are a group of standard buttons in the Core game you can show to create events, ask questions. Each event has a handler function that is used to handle the result of clicking on the button

Events
For information on events and eventid's please see Slave - Events - Common.html

General Structure
Each button has
  • a Show function to display the button and populate variables
  • a Handler to process the event when the button is clicked. These are called for the current slave in training and the assistant
  • a Post Handler called after the Handler finishes processing. These are called for the current slave in training and the assistant
  • an optional Hide function to hide the button
Next Button
There are several buttons labeled Next in the game and each has different handlers and functions to show. The buttons look identical and are in the same position on screen, but are logically different. Each will be separately discussed below.

Hiding Buttons
You will never have to explicitly hide a button in the game. If you choose to show a different button or question the game automatically hides all other buttons first.
But hide functions are supplied as there are some rare situations where you might wish to. For instance in the game there is a situation
  • show an event + text, but no button
  • start a timer that after a few seconds changes the image and shows a button
These are fairly rare situations.

There is a general hide button function that hides all buttons.
function HideButtons();
<HideButtons/>

eg
actionscript:
_root.HideButtons();
xml:
<HideButtons/>

Otherwise see specific functions below to hide individual buttons, not all have separate functions.

Event Next Button

This is the button that is shown during a general event in the game and is usisally shown between parts of a multiple stage event. It will not continue the progression of training (ie move from one act to the next) or similarly for things like contests, end game handling.

The Event Next button does not have a hide function.

Show Function

DoEvent

actionscript:
function DoEvent(eventid:Object)
XML:
<DoEvent>eventid</DoEvent>

Shows shows the Next Event button and when clicked will be intercepted by DoEventNext or via <Events> node.

The eventid parameter is optional for actionscript and will show the button. The event to be triggered when clicked will be in one of these variables
_root.NumEvent/currentnumevent
_root.StrEvent/currentstrevent
NumEvent/currentnumevent is a numeric event, and StrEvent/currentstrevent is a string event. If StrEvent has a value then it is triggered, otherwise NumEvent will be triggered. This is an unusual case and seldom used in the game, mainly for some complex gender based cases and for your slave to override and bypass the standard event.
eg
 actionscript:
_root.DoEvent(400);
eg
 _root.NumEvent = 100 + type;
_root.DoEvent();
 eg
_root.DoEvent("Snakes" + _root.slrandom(2));
eg
_root.DoEvent("LamiaEscape");

xml:
<DoEvent>400</DoEvent>
 <DoEvent>LamiaEscape</DoEvent>

Handler

DoEventNext

actionscript:
slave
function DoEventNext() : Boolean
assistant and event
function DoEventNextAsAssistant() : Boolean
XML:
add a node to <Events>

This handler is called once the Event Next button is clicked, and should check the _root.NumEvent, _root.StrEvent variables.

Return true if you handle an event and false if you do not. There is nothing stopping you from handle standard events in your functions, and it this way you can modify or replace standard events. See Slave - Events - Common.html for a list of standard events.

eg
see below

Special Case:

There are some cases where you want to trigger an event to immediately happen from one of your custom events. This allow you to prefix some effects or text before an event and to use a common event to prevent duplicate code

actionscript:
Here just call the core game version of the handler, passing the event as a parameter.
eg
_root.DoEventNext(400);

XML:
Here there are two variations
eg
<DoEvent now='true'>RanmaChange</DoEvent>
or
<DoXML>RanmaChange</DoXML>
Thse function slightly differently, the <DoEvent> still invokes the standard intercept handlers, allow assistants and events to handle the event. DoXML immediately calls the event, with no other handling. In most cases use <DoXML>

Post Handler

AfterEventNext

actionscript:
slave
function AfterEventNext()
 assistant and event
function AfterEventNextAsAssistant()
XML:
<Events>
    <AfterEventNext>

This handler is always called after the DoEventNext() handler returns. You generally do not need to implement this for custom events. It is mainly used to customise core game events, say adding text or triggering an event.

The core variables
_root.OldNumEvent/lastnumevent
_root.OldStrEvent/laststrevent
are set for the eventid that was processed.

Note: no return type, just process as needed and return;

eg
see below

General Example

actionscript:
In a event or act
_root.AddText("#slave notices something....");
_root.DoEvent(300);
...
function DoEventNext() {
   if (_root.NumEvent == 300) {
       _root.SetText("a monster attacks....");
       return true;
   }
   if (_root.StrEvent == "RanmaChange") {
      _root.SetText("Ranma changes.....");
      return true;
   }
   return false;
}

function AfterEventNext() {
    if (_root.OldNumEvent == 3.4) {
        // love confession, SM answered unsure
       _root.AddText("\r\r#slave looks almost relieved.");
    }
}


xml:
in an event
<AddText>Ranma feels odd....<AddText>
<DoEvent>RanmaChange</DoEvent>
.....
<Events>
     <RanmaChange>
           <SetText>Ranma changes.....</SetText>
     </RanmaChange>
     <AfterEventNext>
           <if lastnumevent='3,4'>
                <AddText>\r\r#slave looks almost relieved.</AddText>
           <if>
     </AfterEventNext>
</Events>

Yes/No Buttons

There are a pair of Yes/No buttons that can be shown. By default these appear at the botton of the General Text window (bottom center of the screen). They can be positioned after the last displayed text if desired.

Each button has its own handler and post handler, but a single show function.

Show Functions

DoYesNoEvent

actionscript:
function DoYesNoEvent(eventid:Object)
XML:
no implementation, see DoYesNoEventXY

Shows shows the Yes and No buttons at the bottom of the general text area (bottom centre of the game screen).

When clicked a different handler is called for each of the yes/no buttons, see DoEventYes and DoEventNo.

DoYesNoEventXY

actionscript:
function DoYesNoEventXY(eventid:Object)
XML:
<DoYesNoEvent>eventid</DoYesNoEvent>

Shows shows the Yes and No buttons immediately below the last text that has been dosplayed. It will show either in the general text areal or larger text area. If both are full (or scrolled) then the buttons will show below the larger text field. This positioning is automatic.

For actionscript the same handlers are used as DoYesNoEvent.

For XML the button click is handled by processing an event node.

For string eventids
eventidYes
or
eventidNo

For numeric eventids
CoreEventYes-eventid
or
CoreEventNo-eventid
eg
 actionscript:
_root.DoYesNoEventXY(400);
 eg
_root.DoYesNoEventXY("HelpSlave");

xml:
<DoYesNoEvent>400</DoYesNoEvent>
 <DoYesNoEvent>HelpSlave</DoYesNoEvent>

PositionYesNo

actionscript:
function PositionYesNo()
XML:
no implementation, automatically done

Repositions the Yes/No buttons to a position immediately after the last displayed text, the same way as DoYesNoEventXY does. This is intended for the limited cases where where you may wish to alter some text AFTER the yes/no buttons have been shown. For instance altering planning/hint text.

ShowYesNoButtons

actionscript:
function ShowYesNoButtons()
XML:
no implementation

Shows the yes/no buttons after they have been hidden. Very, very limited situation where this would be used, generally use DoYesNoEvent. On rare cases it is used in DoPlanningActions.

Questions

You can ask the player questions about their choices in the game or their slaves. These are a list of text options. The list is unlimited in lenght, you can ask as many questions as you like. There is a legacy function AskHerQuestions that is limited to 5 questions, but the other method using ResetQuestion(), AddQuestion(), ShowQuestions() is unlimited. XML us unlimited always.

Questions normally appear after the last text displayed, either in the general text area or larger text area. They can also be optionally shown in the main window, with a white translucent background.

Show Functions

Legacy method

AskHerQuestions

actionscript:
function AskHerQuestions(Event1:Object, Event2:Object, Event3:Object, Event4:Object, Event1Label:String, Event2Label:String, Event3Label:String, Event4Label:String, Caption:String, Event5:Object, Event5Label:String)
XML:
no implementation as such, see below

Event1-5
         event to be done when clicking on one of the four choices. Set to 0 hide the option. Event5 is optional
Event1Label-Event5Label
         the string shown for the question, eg "Runaway screaming" - Event5Label is optional
Caption
         Caption at the top of the questions, eg "What does she do?"
Other buttons are hidden
eg
_root.AskHerQuestions(301, 302, 303, 0, "Please fuck me", "No way!", "Well, I'd prefer it in the ass", "", "How does she respond?");
         // events 301, 302, 303 will need to be handled in the slavegirls DoEventNext() function
_root.AskHerQuestions(301, 302, 303, 0, "Please fuck me", "No way!", "Well, I'd prefer it in the ass", "", "How does she respond?", 304, "How about my mouth?");


Preferred method

actionscript:
To show questions
  • call ResetQuestions() to empty ths list of questions
  • call AddQuestion() one or more times to add a question to the list
  • call ShowQuestions() to show the list

ResetQuestions

actionscript:
function ResetQuestions()

All questions are reset and the questions are hidden.

AddQuestion

actionscript:
function AddQuestion(eventid:Object, label:String)

Add a question for the eventid with the label shown.

ShowQuestions

actionscript:
function ShowQuestions(caption:String, main:Boolean) : Number

Display the question list with a caption specificied. If main is set to true then the question list is shown in the main window. Otherwise below the last displayed text.

Returns the number of questions shown.

PositionQuestions

actionscript:
function PositionQuestions(lines:Number, main:Boolean)

Repostions the questions, this is to allow for the case where the game shows some questions and then you choose to alter the displayed text.

If main is set to true then the question list is shown in the main window. Otherwise below the last displayed text.
lines parameter is normally for internal use.

General Example (actionscript)

_root.ResetQuestions();
_root.AddQuestion(300, "Fuck me");
if (_root.CheckBitFlag2(12)) _root.AddQuestion(301, "Fuck my ass");
_root.AddQuestion(302, "Fuck " + _root.ServantName);
_root.ShowQuestions("She asks you to...");

To modify an existing list of questions, say in training or an event, just use AddQuestion/PositionQuestions
root.AddQuestion(320, "Fuck everyone");
_root.PositionQuestions();

XML:

<Questions>

xml:
<Questions caption='what happens' main='false'>
       <Question event='eventid'>question text</Question>
       <Question event='eventid'>question text'</Question>
</Questions>

Displays the list of questions, same as for AddQuestion() and ShowQuestions() above.

The <Question> nodes can be made optional using <if> <ifor> <ifand> and <else> statements, see example below.

<AddQuestion>

xml:
<AddQuestion event='eventid'>question text</Question>

Use this to add a question to a previously shown list of questions. The list is automatically repositioned.

General Example (XML)

<Events>
    <MyEvent>
          <AskQuestions caption='What does #slave talk about?'>
              <Question event='BigBreastedSlaveComfort'>Comfort her, slave life is good</Question>
             <Question event='BigBreastedSlaveFree'>Promise to help when #slaveheshe can</Question>
          </AskQuestions>
    </MyEvent>

    <BigBreastedSlaveComfort>
            <SetText>comfort her....</SetText>
    </BigBreastedSlaveComfort>

    <BigBreastedSlaveFree>
            <SetText>free her....</SetText>
    </BigBreastedSlaveFree>
</Events>

optional questions
<Events>
    <MyEvent>
          <AskQuestions caption='What does #slave talk about?'>
              <Question event='BigBreastedSlaveComfort'>Comfort her, slave life is good</Question>
             <Question event='BigBreastedSlaveFree'>Promise to help when #slaveheshe can</Question>
             <if customflag1='5'>
                   <Question event='BigBreastedSlaveFuck'>Fuck her!</Question>
             <else>if</else>
                   <Question event='BigBreastedSlaveChat'>Chant with her</Question>
             </if>
          </AskQuestions>
    </MyEvent>

Act Buttons

The act buttons are shown when selecting trainings for your slave. These change be changed, hidden and shown. More detail on this will be given in Slave - Events - Common.html

The acts buttons can be enabled/disabled and renamed

State Change

SetActState

actionscript:
function SetActState(type:Number, tick:Boolean, available:Boolean, actlabel:String, cost:Number, aduration:Number, shortcut:String, astart:Number, aend:Number)
XML:
<SetActState act='act' name='name' available='true' ticked='false' cost='cost' duration='duration' show='true'/>
type - the act number to change. see Slave - Planning.rtf
tick - is the large tick shown, normally to indicate the act is done
available - if false a large slash will indicate you cannot do it
actlabel - the button label, html syntax. You must use the existing shortcut
cost - the GP cost to do the action
aduration - the time to do the action, in hours (1.5 for 1 hour 30 minutes)
shortcut - a single letter for the shortcut to use
astart - cannot be scheduled before this time
aend - cannot be scheduled after this time
pass undefined or onit the parameter to not change the default.

Show the act button. For XML the act attribute can be either the act number or name.
eg
<ShowAct act='7' name='Root her' available='true' ticked='false' show='true'/>

Note: the xml version does not support changing the shortcut key, and really you should never change it.
