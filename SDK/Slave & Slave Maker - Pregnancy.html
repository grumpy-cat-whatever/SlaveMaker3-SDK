General Concepts

There are a series of functions and variables that affect pregnancy in the game. In general these are common to all slaves, assistants and for your Slave Maker with slight variations. The race of the slave (if set) can vary the gestation time, fertility and number of children that result from pregnancies.

Fertility

All slaves/slave makers have a fertility that defines the chance they will become pregnant, by default this is 20%

Contraceptives

Using the Potions dialogue you can choose to have a slave on or off of contraceptives. It is important to note that the Tentacle Beasts are extraordinarily potent and totally ignore these contraceptives.

Progression of Pregnancies

The general process will be

1) your assistant will notice the slave is gaining weight

2) If you own the minor slave Cora she can report that the slave is pregnant in an early morning event or if you talk to her.

3) once the pregnancy comes to term the slave will give birth in a morning event. By default only one slave per day will give birth

Midwives

The minor slaves Cora and Maran are Midwives or Nurses and can supervise the birthing process. They can report on if a slave is pregnant in 2) above. The game will preferentially use Cora for this if she is owned.
Any slave can have this set in their xml via the node <Trainings>

Slave and Slave Maker Variables

These veriables are common to all slaves and also to the Slave Makers
Actionscript Variable XML Variable

PregnancyGestation

 pregnancygestation
pregnancygestation-slavename
(Number) state of the slave's pregnancy
0 = not pregnant with something
 > 0 she is pregnant and this is the number of days until she gives birth
 -1 = will not become pregnant, ever
-1000 = will give birth very soon, normally the next morning
Generally do not modify this value, use the Impregnate() function or <Impregnate> statement, but you can if you wish

Note:
pregnancygestation-slavename supports
pregnancygestation-slavemaker
pregnancygestation-assistant
pregnancygestation-maran

eg
actionscript: xml:
if ((_root.PregnancyGestation < 5) {
   _root.SetText("#slave is due to give birth in a few days...");
}		 <if pregnancygestation-slavemaker='5-'>
   <SetText>You are due to give birth in a few days</SetText>
</if>

PregnancyType

 pregnancytype
pregnancytype-slavename
(String) The type/father of the pregnancy. An arbitrary value but the standard values are
"Yours" - your Slave Maker is the father
"Tentacles" - the slave was impregnated by a tentacle beast
"Human" - impregnated by a general/unspecified person
Generally do not modify this value, but you can if you wish

PregnancyCount

 pregnancycount
pregnancycount-slavename
(Number) The number of children that will be birthed
Generally do not modify this value, but you can if you wish

DateImpregnated

 dateimpregnated (READONLY)
dateimpregnated-slavename (READONLY)
(Number) The date the slave was last impregnated. Uses the internal date format, where 1 represents the first day of year 1252

Configuration Settings

The configurations in configuration.xml affect some defaults for pregnancies. The values are listed below

    <!-- COMMON: the following values are common to all slaves and to slave makers -->
    <CommonDefaults>
   
        <!-- Pregnancy -->
        <!-- the default percentage chance of impregnation -->
        <DefaultFertility>30</DefaultFertility>
       
        <!-- base gestation period for humanoid races in days. A random factor is added to determine the actual value
        and altered based on the mothers race. This base value cannot be less than 35, if it is it will be increased to 35    -->
        <BaseGestation>270</BaseGestation>
       
        <!-- allow multiple people to give birth in a given day -->
        <MultipleBirthsPerDay>false</MultipleBirthsPerDay>
</CommonDefaults>
   

BitFlags

There are some bitflags related to pregnancy

BitFlag 1

13 gaining weight reported
14 NOT taking contraceptives
34 SlaveMaker tentacle pregnancy (OBSOLETE)
88 Pregnancy ON needed for the slave to be trained
95 announced or been told of pregnancy

Slave and Slave Maker Functions

CheckImpregnate

 implemented in: _root and Slave and SlaveMaker class
actionscript:
public function CheckImpregnate(type:String, count:Number, gestation:Number) : Boolean
XML:
no implementation

The game will check if the slave can become pregnant and apply a random check and impregnate the slave based on their fertility.
type - the person or thing impregnating them. See PregnancyType above for default values
count - the number of children. This is optional and will default based on parent and race of the slave
gestation - the number of days until birth. Optional and will default based on parent and the race of the slave
eg
 actionscript:
 _root.CheckImpregnate("Yours");
xml:
<if canimpregnate-current="yours">
     <!-- slave is impregnated your child -->
     <impregnate person='current' type='Yours'/>
</if>

Impregnate

 implemented in: _root and Slave and SlaveMaker class
actionscript:
public function Impregnate(type:String, count:Number, gestation:Number)
XML:
<Impregnate slave='slave' type='whatever' gestation='time' count='number'/>

The specified person will become pregnant. Ignores fertility and contraceptives. Does not impregnate males.

UseContraceptives

 implemented in: _root and Slave class
actionscript:
function UseContraceptives(buse:Boolean)
XML:
<SetFlag flag1-14='buse'/>
<SetFlag flagsm-14-'buse'/>

Puts your slave or slave maker on or off of contraceptives, set buse to true for ON contraceptives.

In XML you have to directly change flag1 for a slave or flagsm for the slave maker.


Slave Interface Functions

These functions can be overloaded for a slave to control their pregnancies. These are all optional, default images and events will happen in not implemented.

Note that originally in the game there were only tentacle pregnancies so the game will default to calling the tentacle version of functions if the others are not implemented

ShowPregnancyReveal

ShowTentaclePregnancyReveal


 actionscript:
function ShowPregnancyReveal() : Boolean
function ShowTentaclePregnancyReveal() : Boolean
XML:
<Images>
    <PregnancyReveal>

Show an image of the slave pregnant, return true if an image is shown. ShowTentaclePregnancyReveal is called only for tenacle pregnancies. ShowPregnancyReveal for all other cases.

NOTE: this is only called if ShowPregnancyBirth/ShowTentaclePregnancyBirth return false

XML Images
Images for xml are in the naming format
Event - Pregnancy Reveal 1.jpg

ShowTentaclePregnancyBirth

ShowPregnancyBirth


actionscript:
function ShowPregnancyBirth() : Boolean
function ShowTentaclePregnancyBirth() : Boolean
XML:
<Images>
    <PregnancyBirth>
or
    <TentacleBirth>

Show an image of the slave giving birth, to a tentacle beast or normal child, return true if the image is shown.

This is the image preferentially used by the game. If it is not implemented or returns false then ShowPregnancyReveal is called above.

XML Images
Images for xml are in the naming format

SeePregnant

actionscript:
function SeePregnant() : Boolean
XML:
<Other>
     <PregnancyGainingWeight>

Your assistant will report to you that they notice the slave is gaining weight. This can be used to modify or prevent the notification. Return true to disable the standard text and either do nothing or show your own text.

For xml you can replace the node <PregnancyGainingWeight> and show/do arbitrary xml

Birthing Events

When a slave, assistant or your Slave Maker gives birth a standard set of events handle the birth.

The general process is

Call the XML node for the birth, this is  in the <Other> node (and in Other.xml in the game installation)

<parentPregnancy>
for a slave
eg
<TentaclePregnancy>

For your Slave Maker
<SMparentPregnancy>
eg
<SMTentaclePregnancy>

By default the game implements

Tentacle
Human
Yours
as options and a special case for Shampoo. You can implement these arbitrarily as you can specify any arbitrary string for PregnancyType, so you could potentially have
<RanmaPregnancy>
if you used
<Impregnate type='Ranma'/>
for a slave

Once the event is complete the image is shown by the game.

Note that tentacle pregnancies are a multiple stage event but other pregnancies are a single stage event.
