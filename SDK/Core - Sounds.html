Sound Effects
Slave Maker 3 SDK Updated to version 3.4.02

The game has a number of built-in sound effects, and a number of mp3 files included. Developers can also play any arbitrary mp3 audio file when desired.

Core Variables

Sound effects are enabled in options. Enabled by the setting SoundOn
Actionscript Variable XML Variable

SoundsOn

 soundson
(Boolean) true if sound effects are on
eg
<if soundson='true'>
   <!-- sound effects are enabled -->
</if>

Note:
In practice you will very seldom, if ever, need to use this variable. The standard game functions below all internally test it and do not play any sound if false.

Core Functions

All actionscript functions are contained in a class instance Sounds of the class SoundsModule.
eg
_root.Sounds.Beep();

For legacy reasons most can be accessed without the Sounds. reference. but please do not.

All functions internally test SoundsOn, so you do not need to check it before calling any of these functions.

StartMoaning

actionscript:
function Sounds.StartMoaning(count:Number)
XML:
<PlaySound count='number'>Moaning</PlaySound>

Play the sound of one or more women passionately moaning, count is the number playing in parallel, default 1.

 eg
 actionscript:
_root.Sounds.StartMoaning(2);

 xml:
<PlaySound count='2''>Moaning</PlaySound>

Note:
can play simultaneously with StartFucking()

PlaySound

actionscript:
function Sounds.PlaySound(snd:String, repeats:Number, count:Number, volume:Number, delay:Number)
XML:
<PlaySound repeats='number' count='number' vol='number' delay='number'>snd</PlaySound>

Play a standard sound or an external mp3 file

         snd - either
                  a path to an mp3 file to play (eg "Sounds/Circus.mp3")
                  the name of a standard sound effect, see below (eg "DoorClang")
         repeats - the number of times to sound effect will play, only applies for mp3 files and selected sound effects
         count - special parameter for "Moaning" and "Fucking" it is the number of similtaneous effects, ignored otherwise
         volume - the volume level, 0-100. Note does not always work, and it is a global level, it cannot be set for individual sounds if you play them simultaneously
         delay - a delay in milliseconds before the sound plays

         Note: pass undefined for a parameter you do not wish to use in actionscript. In xml just omit the attribute.

 eg
 actionscript:
_root.Sounds.PlaySound("Sounds/Circus.mp3", 1000);
_root.Sounds.PlaySound("Frying");

xml:
<PlaySound>Sounds/Circus.mp3</PlaySound>;

StopSounds

actionscript:
function Sounds.StopSounds()
XML:
<PlaySound stop='true'/>

Stops the playback of all sound effects

 eg
 actionscript:
_root.Sounds.StopSounds();

 xml:
<PlaySound stop='true'>

Note:
Automatically called by the game after each act/training and by calling _root.HideImages(); or <HideImages/>

StopAndFadeSounds

actionscript:
function Sounds.StopAndFadeSounds()
XML:
<PlaySound stopandfade='true'/>

Stops the playback of all sound effects but fades them out instead of stopping abruptly.

 eg
 actionscript:
_root.Sounds.StopAndFadeSounds();

 xml:
<PlaySound stopandfade='true'>

Standard Sound Effects

These are embedded sounds that can be used with PlaySound function above.

eg.
_root.PlaySound("Frying");

Some play once, some loop permanently. The game will stop them automatically when you/the game calls _root.HideImages(), which is automatically done at the end of an act/training, commonly when the Next button is pressed. You can explicitly stop the sound effects with StopSounds()
