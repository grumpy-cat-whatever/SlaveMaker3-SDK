Creating an Actionscript based Slave

Slave Maker 3 SDKVersion 8

Slaves can be developed using either Adobe Flash or the free FlashDevelop program. There are differences in image displaying between the two and some limitations in one case.

For FlashDevelop, you can also use the code generator SlaveBuilder to construct the base project and source code.

Adobe Flash CS3 or later
OR
FlashDevelop (current version)

Graphic editing program, I use ImageJ and Paint.Net, two quite good free editors.

Common Setup for both Adobe and Flash Develop


1) Name the slave

Pick an unused name for the slave. For instance you cannot use Rei as that is taken.

2) Collect Images

Collect all images needed for the slave, see the Image List. Pick an image to appear in the slave market

3) Create an image for the slave to appear in the slave market

Add an image like
Slaves/Slave-name.png

This can be a jpg, png. gif file, recommend jpg or png. This image must have a transparent background. If you do not know how to do this, post in a thread in one of the forums and someone will be able to edit it for you

Note the name must correspond exactly to the name of the swf file to be created below.

4) Create the SlaveGirlXX.txt file

Create the file
Slaves/SlaveGirlxx.txt
where xx is a number. During development just use a free number less than 200 and edit new contents. Once you are ready for a release a number will be assigned to you..

eg
girlname=Ifurita&girldesc=Ifurita : This girl is lost, claiming to have travelled many dimensions.&image=Slave-Ifurita.png&gamefile=Slave-Ifurita.swf

This is a set of url encoded variables. Some are optional and can be left out, some are mandatory. These variable names are case-sensitive, and they are

girlname
The name of the slave
girldesc
A short description of the slave to appear in the Slave Market
image
the relative filepath for the image to show in the slave market. It should be a png file with a transparent background.
This file MUST be named like
Slave-slavename.png
gamefile (optional)
This is the name of the swf file for actionscript based slaves. As of version 3.3 this is optional for xml slaves
Dickgirls (optional)true/false - are dickgirls a mandatory part of this slaves training.
Tentacles (optional)true/false - are tentacles a mandatory part of this slaves training.
Furries (optional)true/false - are furries a mandatory part of this slaves training.
renown (optional)none              = no slavemaker reputation needed (0) - default
         low               = reputation 10 needed
         medium   = reputation 30
         high              = reputation 60
         max               = reputation 100
mode (optional)a general guide to how difficult the girl is to train
         easy             
         normal            (default)
         hard
Pregnancy (optional)true/false - is pregnanc a mandatory part of this slaves training.
NonHuman (optional)true/false - are non-humans (and sex with them) a mandatory part of this slaves training.
Folder (optional)a folder that this slave will store it's images in. This is optional, and can be defined in the slave's xml file
available (optional)true/false - if false the slave is not visible in the slave market and is only shown be a special event when training another slave or a core game event
vanilla (optional)true/false - if true the slave has no custom events or trainings
assistant (optional)true/false - the slave is initially an assistant. This can change due to events.
slavetype (optional)
untrained, minor, assistant

Note for foreign languages you can also create files like
SlaveGirlxx-French.txt
and so on. These will contain the translated text.

5) Create the xml file (optional)

Create an xml file for the slave
Slave-slavename.xml

At a minumum this will have the name, version, credits for the slave

<Language>
    <Slave>
        <Version>2</Version>
        <Credits>Developed originally by Pooloka, continued by cmacleod</Credits>
        <Name>Yurika</Name>       
    </Slave>
</Language>

This file can contain any xml nodes and code as needed. See XML slave development for more details, and Ranma's files for a simple actionscript/xml hybrid.

6) Create an image folder for the slave (optional)

All slaves canl have a folder in a location
Images/Slaves/name
eg
Images/Slave/Mai


Common Tips for Programming a Slave


First and foremost use ActionScript 2, do not use the more recent Actionscript 3.

My strong guideline for girls is
1) no black and white images
2) no speech bubbles or sound effect text
3) decensor all images
4) any image dimension above 640x480 is fine, I commonly use 800x600 or 1068x800. The main window is 4x3 aspect ratio (448x336 pixels)
5) provide at least two variants for all sex acts, actually for all acts where possible


A couple of general notes on programming flash

a) all functions must have less than 500 lines of code, You can get very odd things happen if you do not. This is purely what is in
function xxxxxx()
{
}
not the total lines in an actionscript file

b) I have seen reports that switch statements should have less than 20 terms as you can get odd issues. This may be related to a)

c) Careful of floating point arithmetic. The variable of type Number is a floating pont number. So you can get cases where 0.1 + 0.1 + 0.1 + 0.1 != 0.4

d) Stats like
VarObedience
are not integer and do not include dress and jewelry, makeup etc effects. To use the integer versions, with the items included use
VarObedienceRounded
or to round down
int(VarObedience)
or round up
Math.ceil(VarObedience)

d) Careful of typos. The compiler will not tell you that a function or variable you use does not exist
eg
AddTexT("hi")
compiles fine but does nothing at run-time

Similarly using a wrong variable
_root.VarOBedience
works fine but will have a value undefined

Similarly for any object, movieclip etc
Also the same if you leave off _root. when accessing a global/base object

e) Event numbers as used in _root.NumEvent, if you use an invalid one the game will generally hang (next button will do nothing)

Using Adobe Flash

To create your own slave using Adobe Flash

1) Create the base project by copying an example from this SDK

In the SDK, folder
Examples/Slave Girl/Base Girl
rename the files
Slave-Girl.fla
Slave-Girl.as
to match the name of your girl. For ease from now on I'll assume the girl will be named Ifurita. So rename to
Slave-Ifurita.fla
Slave-Ifurita.as
Provided are also example SlaveGirlXX.txt and Slave-Girl.xml files.

2) Edit the project to customise for your slave

Open the fla file using Adobe Flash. Please note this will not be a tutorial on how to program flash. On the top layer in the timeline (labelled ActionLayer) edit the actions of frame 1 and change the include statement to the new actionscript file Slave-Ifurita.as
eg
#include "Slave-Ifurita.as"

3) rename layers in the library and stage (optional)

Rename the folder Slavegirl to Ifurita if you like

4) Update images and add any new ones needed

Edit images and action script.

5) Update actionscript for events etc


6) Publish

Publish and copy the swf, png, txt files into your game installation


Using Flash Develop

Important Restriction

for Actionscript 2 only supports up to 16Mb of images. You must limit yourself to that size of embedded images. If you want to exceed this limit you will have to dynamically load images at runtime using funcntions like AutoLoadAndShowMovie()

Class Based Slaves Only

All slaves developed using Flash Develop must use a structure based on a class that extends the SlaveModule class. FlashDevelop does not support global functions and variables as Adobe does.
