Using the Debugging Menu

For testing of slaves the debugging menu in the game is very useful. This is opened by holding Ctrl key and the Shift key and simultaneously clicking the letter G in the Gold text in the lower left of the screen using the mouse. This will add a new tab, click on it and you have the following

Dress Layout and Items

You have the ability to position dress items using drag and drop or typing the coordinates explicitly. The xml and actionscript code for the item is how.

To use this click the button "Load Items" then type an item number in the input box "Show Item". The item will then be shown and you can drag it into place and type the coordinates. Note
- negative width/height reverse the image
- rotation is in degrees, can be negative
Use the button "Clear Items" to remove the items.

You can use the Dress +/- and Naked buttons to select the dress or naked image to select. The frame +/- buttons select an alternative (notably for naked images)

Tint field can apply a tint to the current item, click the colour and a simple colour picker will be shown

View Slave Details

The Slave button at the top lists all slaves, includes details on pregnancy. By default this show all slaves and assistant, including ones you do not own (have sold or are not employed) You can filter this to only owned slaves by holding the Shift key as you click the button
The number like Slave 14: is the index into the SlavesArray for the slave (the SlaveIndex variable) and is also the index for the slave in the smxslaves.sol file
Type is the type of slave, 0 minor slave, -1 assistant, -10 current slave. Details in the SDK

View Flag Details

View all flags like BitFlag1, BitFlagSM and the various flags for events

View People/Places

similarly view the flags and variables for all people and places in the game

Check any Arbitrary Variable Value

In the white text area (clear it in needed first) type
?variable
and click on the "Evaluate" button
eg
?constitution
this will display the value of the constitution stat for the current slave.

This supports
- xml variable naming
- absolute actionscript variable name
eg
?NumMerchant
will display the variable
_root.NumMerchant

Run any Arbitrary XML code

You can type any arbitrary xml code in the white text area (multiple statements fine) and click the "Evaluate" button. The game will execute the code and any output (say using <AddText> will be shown in the same area

Other buttons

+1 Day - move time forward 1 day
+1 Hour - move time forward 1 hour
Love - displays love specific information for all slaves
Reset - will reset people/places and events back to default. Basically intended as a attempt to fix errors in data
Log - not very useful yet. some key statement log output here. Also if you use the xml statement <trace>whatever</trace> or _root.SMTRACE("whatever"); ONLY logs once the debugging menu has been enabled
View XML - used to list xml nodes for debugging some issues. Type say in the white area
Slave
and it will list all xml nodes in <Slave>
Slave1/Slave2 - used for twins as each twin can have a separate dress image and thus separate item positioning

Other Fields

The gender fields allow you to immediate change the gender of the slave or slave maker
The catgirl etc field allows you to make the slave immediately a catgirl, type 1 for YES and 0 for NO
FairyXF is a ratio 0-100 and is the alpha transparency of the wings
