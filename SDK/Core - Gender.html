Gender Concepts

All slaves, assistants and your slave maker have a gender (well clearly) and these are specified by a number

0 = no gender/neutral gender
1 = male
2 = female
3 = dickgirl
4 = male twins
5 = female twins
6 = dickgirl twins
7 = mixed twins
       
As you see there is a general pattern, values > 3 are twins, and for those the value - 3 is the standard male, female, dickgirl. Note the value 7 for mixed twins breaks this pattern though. Note that Slave Makers are only allowed values of 1, 2 ,3, twins are not permitted.

All slaves and your slave maker have a current gender and the gender they were when they were born.

Dickgirls or hermaphrodites can be disabled by a game option.

XML

Using XML the following text can be used as well as the numbers to test or set any gender variable
     "none"
    "male"
    "female"
    "dickgirl" or "hermaphrodite" or "futanari"
    "maletwins"
    "femaletwins"
    "dickgirltwins"

When testing in a conditional you can additionally use
     "feminine"  - a female or dickgirl
     "masculine"  - a male or dickgirl
     "twins"  - any of the types of twins, numbers 4+

Gender Identity

Some slaves can choose to refer to themselves as a different gender. For instance a male 'trap' who refers to themselves as 'she'. Similarly a tomboy referring to themselfs as 'he'. The last is a bit more common in Japanese culture.

There are two game options to cover this
Effiminate Characters
There are some characters who appear to be female but are actually cross-dressing males,. In this case the game has a flag, BitFlag1, flag 62 that means the game will descibe them as female and act decriptions will use the female version of descriptions
Gender Identory
Each slave has a variable which is the gender they identify with. When using termis like he/she/his/her the game will use this identity, not their actual gender. This identity defaults to their actual gender, but can be changed at start of game or during game play.

Variables

Slave Variables

The following variables are valid for all slaves, assistants amd minor slaves in the game. For the current slave in training the variables are copied into global scope.

Note for XML, the following lists
XML Variable - the variable you use the access or change during gameplay
XML Initialisor - The node you add to <Slave> node to initialis the variable when the slave is created
eg
<if slavegender='1'>
    <!-- slave is male -->
</if>

In the slaves xml
<Slave>
    ....
    <Gender>male</Gender>
</Slave>


Actionscript Variable XML Variable
XML Initialisor
SlaveGender
implemented in: _root, class Slave via SlaveStatistics
 slavegender
<Gender>
(Number) The gender of the slave (NOT for Slave Makers), default 2. See Gender Concepts for details of values.

 Please use ChangeSlaveGender() or the xml version to change a slaves gender.

eg

if (_root.SlaveGender == 2) {
   // current slave is female
}
var sd:Slave = _root.SMData.GetSlaveDetails("Tifa");
if (sd.SlaveGender == 2) {
    // slave Tifa is female
}
xml:
<if slavegender='2'>
    <!-- current slave is female -->
</if>
<if slavegender='male'>
    <!-- current slave is male -->
</if>
<if slavegender-ranma='2'>
    <!-- slave Ranma is female -->
</if>

Assistant Variables

The assistant has all Slave variables above in the _root.AssistantData object, but for general use the following variables are also defined in global scope.

Actionscript Variable XML Variable
ServantGender (READ ONLY)
implemented in: _root
 assistantgender
(Number) Your current assistant's gender, default 2, their actual, permanent gender, See Gender Concepts for the standard values..

Do not alter the actionscript variable ServantGender, it is a read only variable. Please use ChangeSlaveGender()  or the XML equivallent to change an assistants gender.

Core Variables

Actionscript Variable XML Variable
Owner.PersonGender
implemented in: _root and class PersonOwner 		ownergender
The gender of the current slave's owner. Defaults to 1.

PersonGender (READ ONLY)
implemented in: _root and class Personr 		persongender
partnergender
During events or sex acts where another person is present this variable is set to the gender of the other person. This includes cases like Owner Testing and End Game, where it is set to the gender of the slaves owner. If the other person is your Slave Maker or assistant it is set to their gender.

So for example in the sex act 69. If you select to do it with the slave and your Slave Maker it will be set to your Slave Makers gender. If you select a slave, then it is set to the slave's gender.

In xml you may interchangably use persongender or partnergender, the different names for context, a partner is used for sex acts for instance.
PersonOtherGender (READ ONLY)
implemented in: _root and class Personr 		personothergender



Core Functions

Gender Changes

ChangeSlaveGender
 implemented in: _root and Slave class
actionscript:
function ChangeSlaveGender(gnd:Number)
   gnd only accepts the numeric gender values.
XML:
<SetVar slavegender-slave='gender'/>
   -slave is optional, see below for details

Change your slave's gender to the specified value. Updates all game areas and displays for the change

 eg
 actionscript:
eg
_root.ChangeSlaveGender(3);

Class Slave version (for an arbitrary slave, say a minor slave)
var sd:Slave = _root.GetSlaveDetails("Maran");
sd.ChangeSlaveGender(3);

 xml:
<SetVar slavegender='male'./>
<SetVar slavegender='3'/>
<SetVar slavegender-Maran='dickgirl/>
note: -Maran means change the gender of the slave Maran.

ChangeSlaveMakerGender
actionscript:
function ChangeSlaveMakerGender(gnd:Number)
   gnd only accepts the numeric gender values 1, 2, 3
XML:
<SetVar gender='gender'/>

Change the Slave Maker's gender to the specified value, values 4+ are converted to a value 0-3

 eg
 actionscript:
eg
_root.ChangeSlaveMakerGender(3);

 xml:
<SetVar gender='male'./>
