Catgirl Training
Slave Maker 3 SDK Updated to version 3.4.02

There are a series of functions and variables that control catgirl training for your slaves. Note males can do catboy training but for simplicity I will refer to catgirls from here on.

All images are stored in a movieclip
_root.CatgirlsMovie
all code is stored in a module
_root.Catgirls

There are very limited situations where you will refer to these but there is an event related function you can use and a variable you can set.

Core Variables


Actionscript Variable XML Variable

Catgirl (READ ONLY)

implemented in: _root
 iscatgirl (READ ONLY)
(Boolean) Is the current slave in training a catgirl, if set to true they are a catgirl. This is equivalent to using the IsCatgirl() function. XML version can be used for any arbitrary slave using iscatgirl-slavename

eg
actionscript: xml:
if (_root.Catgirl)
   // they are a catgirl
} else {
   // not a catgirl
}
OR
if (_root.IsCatgirl()) {
}
 <if iscatgirl='true'>
   <!-- catgirl -->
<else>if</else>
   <!-- not a catgirl -->
</if>
<if iscatgirl-aeris='true'>
    <!-- slave areis is a catgirl -->
</if>

sCatTrainer

implemented in: _root, classes SlaveMaker and Slave via SlaveMakerSkills
 cattrainer
(Number) your slave makers skill as a catgirl trainer. Also assistants and slaves have this skill and can be checked via their Slave object
XML can use cattrainer-slavename to get the value for an arbitrary slave/assistant. This defaults to your current slave maker.

eg
actionscript: xml:
if ((_root.sCatTrainer > 1)
   // slave makers cat trainer 2 or higher
}
var sd:Slave = _root.GetSlaveDetails("Melisandra");
if (sd.sCatTrainer > 2) {
    // Melisandra's cat trainer 2 or higher
}
if (_root.AssistantData.sCatTrainer > 2) {
  // assistants skill 2 or higher
}
 <if cattrainer='2+'>
   <!-- slave makers cat trainer 2 or higher -->
</if>
<if cattrainer-melisandra='2+'>
   <!-- Melisandra's cat trainer 2 or higher -->
</if>
<if cattrainer-assistant='2+'>
   <!-- sassistants cat trainer 2 or higher -->
</if>

slCatTraining

implemented in: _root, class Slave via SlaveSkills
 cattraining
(Number) The level of training as a catgirl for a given slave.
         -1          = will never train
         0           = not started
         1-99     = partial
         2000    = fully trained    
Generally do not directly refer to this variable, use the ChangeCatTraining(), IsCatgirl(), IsCatgirlComplete() or IsCatgirlTraining() functions.

Note:
The maximum value of this skill is determined by the Slave Makers sCatTrainer skill. Level 1 means a maximum of 60, Level 2 is unlimited.

CatgirlInterest

implemented in: _root, class Slave
 catgirlinterest
(Number) The level of interest your slave has in being a cat girl. This is the skill level of sCatTrainer needed to start training. It is NOT saved so you must set it in Initialise() or UpdateSlave() functions and when it changes
         0 = always will train if sCatTrainer is 1 or more
         1 = will train is sCatTrainer is 1 or more
         2 = will train is sCatTrainer is 2 or more
Special case
         3 = will never train. Preferably do not use this value, generally set to 2 to indicate high resistance.

CatGirls.CatgirlEvents

mplemented in: _root
 iscatgirlevent (READ ONLY)
(Boolean) if set to false then standard catgirl events are suppressed for the current planning action (sex act, chore, job etc), Mainly use in a ShowJobAcolyte etc function to replace or totally change catgirl events.

For XML you can only test the value at this time.

Core Functions

ChangeCatTraining

 implemented in: _root and Slave class
actionscript:
function ChangeCatTraining(val:Number)
XML:
<ChangeCatTraining slave='person'>val</ChangeCatTraining>

Change catgirl training by val points. By default the current slave in training is updated.

 eg
 actionscript:
 _root.ChangeCatTraining(5);
var sd:Slave = _root.GetSlaveDetails("Tifa");
sd.ChangeCatTraining(5);
xml:
<ChangeCatTraining>5</ChangeCatTraining>
<ChangeCatTraining slave='Tifa'>5</ChangeCatTraining>

IsCatgirl

 implemented in: _root and Slave class
actionscript:
function IsCatgirl() : Boolean
XML:
iscatgirl

Returns true if your slave is a fully or partially trained catgirl. Available both via _root (for current slave) or via Slave class.

 eg
 actionscript:
 if (_root.IsCatgirl()) {
   // current slave is a catgirl (or training as one
}
var sd:Slave = _root.GetSlaveDetails("Tifa");
if (sd.IsCatgirl()) {
    // Tifa is a catgirl
}
xml:
<if iscatgirl='true'>
    <!-- current slave is a catgirl -->
</if>
<if iscatgirl-tifa='true'>
    <!-- Tifa is a catgirl -->
</if>

 implemented in: _root and Slave class

XML:
iscatgirlcomplete

Returns true if your slave is a fully trained catgirl, returns false if they have not completed training. Available both via _root (for current slave) or via Slave class.

 implemented in: _root and Slave class

Returns true if your slave has started training as a catgirl but has not yet completed training. Available both via _root (for current slave) or via Slave class.

XML:

eg

           <BitGag frame='4' x='220' y='79' rotation='-10' width='55' height='40'/>
           <NymphsTiara x='216' y='47' rotation='-10' width='50' height='45'/>
           <CatEars frame='1' x='222.25' y='52.55' rotation='0' width='50' height='25' over='true'/>
           <Tail x='152' y='226' rotation='0' width='-150' height='80'/>
           <Wings x='217' y='150' rotation='10' width='-300' height='200'/>
    </Dress0>

PositionCatEars
actionscript:
function PositionCatEars(gframe:Number, xpos:Number, ypos:Number, rot:Number, wid:Number, hei:Number)
XML:
<CatEars x='xpos' y='ypos' rotation='rot' width='wid' height='hei' frame='num'/>

Position the cat ears behind the slave's image in the main window. The dress image must have a transparent background for the ears to be seen.

gframe = select a particular version of the ears. A White/pink and Black/white are the standard set.
xpos = horizontal position in the main window, 0-480, but you can exceed these limits to have the image partially out of the window
ypos = vertical position in the main window, 0-336, but you can exceed these limits to have the image partially out of the window
rot = rotate the image clockwise in degrees, < 0 for counter-clockwise
wid = width of the image. Will resize the image, will not preserve aspect ratio
hei = height of the image. Will resize the image, will not preserve aspect ratio

Note: the debugging menu can create these parameters for both actionscript and xml

PositionCatEarsOver
actionscript:
function PositionCatEarsOver(gframe:Number, xpos:Number, ypos:Number, rot:Number, wid:Number, hei:Number)

Position a set of cat ears over top of the slave's image in the main window.
Otherwise identical to PositionCatEars.


Slave Interface Functions

These functions can be overloaded for a slave to control their catgirl training.


These are optional

actionscript:

XML:

Start catgirl training, show an image and optionally specific text
return true if you show an image, otherwise a default image will be shown

ShowRefuseCatgirl

actionscript:
function ShowRefuseCatgirl(type:Number)
XML:
<Other>
    <RefuseCatgirl>
xml variable general is set to the value of type.

type
         23 = wear cat ears
         24 = wear cat tail
         1 to 6 = put on dress x with cat ears or tail

Show an image/text if the slave refuses to start catgirl training. When this is called default text has already been shown.
