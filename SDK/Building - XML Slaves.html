Building an XML based Slave
Slave Maker 3 SDK Version 8

Every slave and assistant has an optional XML file that can be used to initialise their variables, show introductions and even show images.

See Core - XML Events - Common.html for a general overview of xml in the game and file locations.

Creating the XML Slave Files

1) Create a new xml file for the slave

Use the naming system/pattern
Slaves/Slave-name.xml
ie
in the Slaves folder, and where name is the name of your slave.

 For translation copy this file and create for Russian for example
Slave-Aeris-Russian.xml
and similarly for other languages. Note these are in the same folder, not a subfolder like above.

2) Create a new SlaveGirlxx.txt file

See Building - Actionscript Slaves.html for details.

3) Create an image for the slave to appear when selecting

Add an image like
Slaves/Slave-name.png

This can be a jpg, png. gif file, recommend jpg or png.

Note the name must correspond exactly to the name of the xml created above. Also note slaves do not use an <Image> node for this to be shown.

4) Create an image folder for the slave and a base image

All slaves will have a folder in a location
Images/Slaves/name
eg
Images/Slaves/Daisy

Now add an image file to that folder
Assistant.jpg

This is the default image shown in the lower left person window during game play when the slave is used as an assustant. Ideally the image should be square (ie width equal to height), but this is not required.

Create the XML for the Slave

All data in the slave's XML file are structured in general as

<Language>
         <Slave>
         </Slave>
        
         <Events>
         </Events>

         <Planning>
                  <Acts>                    
                           <DoPlanning>                               
                           </DoPlanning>
                  </Acts> 
         </Planning>

         <Other>
         </Other>

         <EndGame>
         </EndGame>

          <Assistant>
         </Assistant>
        
</Language>

All the nodes are optional, except the <Slave> node.

These node represent

<Slave>

This is the initialisation information for the slave. It contains all the statictics, meaurements, skills and similar information for the slave.

<Events>

This node has the custom events for the slave.

Note: this node is included in the <Slave> node for assistants and minor slaves. For a slave you can include versions inside <Slave> as well. This will represent events for the slave when
  • slave is acting as an assistant
  • you talk to the slave via the 'Talk to Slaves' screen
when used as an assistant or owned slave the <Events> node outside of <Slave> node is ignored completely.

<Planning>

This is where you customise the slaves tranings and interactions with your Slave Maker. You can add custom jobs, chores, schools, sex acts. You can also customise talking with your slave makers and the various standard trainings.

As for <Events> yhere is a separate version included inside <Slave> node for use when the slave is an assistant, owned slave or minor slave. Notably for the <SlaveTalk> and <SlaveIntimacy> nodes.

<Other>

This node allow you to customise love confessions and pregnancy effects for the slave.

As for <Events> yhere is a separate version included inside <Slave> node for use when the slave is an assistant, owned slave or minor slave. Notably for the <SlaveTalk> and <SlaveIntimacy> nodes.

<EndGame>

This is where you implement custom endings for the slave

<Assistant>

This node contains specifc nodes for the the slave for when they are used as an assistant

<Slave> Initialisation

All slave variables can be initialised in the <Slave> node. Some are only checked when a new game is started and others also when a game is loaded. The game automatically decides which variables to load at which time. In most cases the name of the tag is the same as an existing variable in this SDK

A few variables are clearly for start game only, particularly those for the introduction screens.
