General Information

1) Your current assistant is either a trained slave or a fellow slavemaker you hire. Trained slaves are free to use and fellow slavemakers have to be hired. Internally these are identical except for some flags. Both are stored as an instance of the class Slave in the array _root.SlavesArray/ A reference to this is kept in the variable AssistantData

2) Other data is held in global assistant variables in _root scope. These are generic flags and text related variables.
eg
_root.ServantName
Mostly I will leave off the _root from now on.

3) Also you will see some variables named Servantxxxxx and some Assistantxxxxxx, this is a legacy from older versions. The standard usage now is Assistantxxxxxxxx.


Assistant Object

actionscript:
A reference object holds the current assistants statistics, items, dresses, skills and the such. This is

var AssistantData:Slave

A copy of the class, and it's dependencies is included in this SDK
Common Scripts/Classes/Slave.as

This object is available at any time
eg
if (_root.AssistantData.VarIntelligence > 50) _root.ServantSpeak("I do not think she will do that.");

XML:
For XML this object is automatically used when needed and no explicit reference is needed. Any variable of flag can refer to your assistant using -assistant
eg
<if charisma-assistant='50+'>
     <PersonSpeak person='assistant'>I do not think she will do that</PersonSpeak>
</if>
If preferred you can also explicitly refer to the assistant by name as you can for any slave
eg where Shampoo is your assistant
<if customflag2-Shampoo='1'>
     <SetVar customflag2-shampoo='2'/>
</if>

Global Variables

The remaining variables are generic ones used, often for historical reasons over the development of the game. A number of these are also stored in the AssistantData object, notably ServantName

Saved


The following variables are saved/loaded by the game, so any changes are tracked by the game.

Actionscript Variable XML VariableText Macro

ServantName

(Slave class) SlaveName
 none directly but can change using
<SetVar slavename-assistant='newname'>
#assistant
(String) The name of your assistant. For twins this is the full name like 'Manya & Minea'

actionscript:
You can also be reference this variable using
_root.AssistantData.SlaveName
BUT if you change one of these variables the other is not changed, so always change both in actionscript. Note the game synchronises these variables when you save/load a game but not at any other time.
eg
_root.ServantName = "Lady Shampoo";
_root.AssistantData.SlaveName = ServantName;

xml:
When using xml the <SetVar> noted will automatically change both.

xml initialisation:
Initialise in <Assistant> node using
<Name>their name</Name>

Macro:
This macro will be substituted with your assistants full name
actionscript:
_root.SetText("#assistant will take your slave....");
xml:
<SetText>#assistant takes a walk</SetText>

ServantName1

ServantName2

(Slave class) SlaveName1 & SlaveName2
 none directly but can change using
<SetVar slavename1-assistant='newname'>
<SetVar slavename2-assistant='newname'>
#assistantA
#assistantB
(String) These are meant for the case of twins, they are the name of one of the twins

Otherwise access and use similarly to ServantName.

xml initialisation:
Initialise in <Assistant> node using
<Name1>twin 1 name</Name1>
<Name2>twin 2 name</Name2>


Not Saved

The following servant variables are NOT saved and should be set when initialising the assistant (or slave when used as an assistant)

AssistantDescription - a description of the assistant and a summary of any effects he/she gives
eg
_root.AssistantDescription = "A wild, erratic woman with arcane abilities and maybe demonic ties.\r\rNymphomania + 5, Fearful when crossed.";

AssistantCost - cost in GP to hire as an assistant, defaults to 0
ServantPronoun - Pronoun, default "I", for Shampoo it is "Shampoo" so she speaks in the third person
ServantGender - servants gender, default 2
         1 = male
         2 = female
         3 = dickgirl
         4 = male twins
         5 = female twins
         6 = dickgirl twins
AssistantTentacleSex - can your assistant be tentacle fucked, default true
AssistantRape - can your assistant be raped, default true
SlaveLikeServant - misleading name, does the servant like the slave, default true (will rename in a later version)
