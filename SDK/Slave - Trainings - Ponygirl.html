Ponygirls in the Game

Ponygirls are uncommon in Mioya and most Slave Makers have no experience with them, and many have never even heard of them. By default Slave Makers are not familiar with ponygirls.

The variable PonygirlAware controls the slave maker's awareness of ponygirls in the game.

There is an invitation only ponygirl race held in the game, it is not publicised and by default a Slave Maker does not know it exists but will learn of it when they learn of ponygirls..

Your slave makers skill sPonyTrainer controls how good a ponygirl they can train. The skill limits the slave's ponytraining level slPonyTraining

Ponygirls are optional in the game and can be disabled in Options. If they are disabled then PonygirlAware will be -1 and the setting PonygirlsOn will be false.

Core Variables

Actionscript Variable XML Variable

PonygirlsOn

 ponygirls
Option to enable ponygirls in the game. Applies to all slaves and slavemakers. Default true which means they are present

PonygirlAware

 ponygirlaware
(Number) awareness of ponygirls
         -1 = no ponygirls in game
         0 = not aware (default)
         1 = aware
Note: generally you should never modify this variable, let the game set based on the standard events.

slPonygirlTraining

 ponygirltraining
(Number) The progress of your slaves training (0-100).

PonygirlInterest

 ponygirlinterest
initialise using <PonygirlInterest>
(Number) The level of interest your slave has in being a ponygirl. This is the skill level of sPonyTrainer needed to start training. It is NOT saved so you must set it in Initialise() or UpdateSlave() functions and when it changes
         0 = always will train if sPonyTrainer is 1 or more
         1 = will train is sPonyTrainer is 1 or more
         2 = will train is sPonyTrainer is 2 or more
         3 = will train is sPonyTrainer is 3 or more
Special case
         4 = will never train. Preferably do not use this value, generally set to 3 to indicate high resistance. Only for special cases

sPonyTrainer

 ponytrainer
(Number) your ability to train ponygirl, 0-3
         0 = do not know how to train them
         1 = can train girls except those hostile to the idea. They are willing but not perfect ponygirls and can only get the ending Pony Slave
         2 = can train girls except those hostile to the idea. They are fully submissive ponygirls and can get the ending Ponygirl
         3 = can convince any girl to become a ponygirl and love it. They are truly the ideal ponygirl and can get the ending Perfect Ponygirl

TotalPony

 total-ponygirl
totalponygirl-slavename
(Number) Total times your slave became a ponygirl/boy (almost always 0 or 1).

WinPonygirl

 winpony
(Number) how many times your slave has won a ponygirl race.

TotalContestPonygirl

 totalcontestponygirl
totalcontestponygirl-slavename
(Number) Total times your slave competed in a ponygirl race.

DifficultyPonygirl

 difficulty-ponygirl
difficultyponygirl-slavename
(Number) Obedience needed to be a ponygirl.

Ending

Ending Number: 18
Condition:      IsPonygirl() && Score > 50 && BitGagWorn == 1 && HarnessWorn == 1 && (PonyTailWorn == 1 || HasTail) && slPonyTraining == (40 + (sPonyTrainer * 20))

Images are shown using the interface function ShowEndingPonygirl, see below.

The condition means 'must be a ponygirl, wearing all gear, maximum ponygirl training and a score of 50+'

Core Functions

IsPonygirl

actionscript:
function IsPonygirl() : Boolean
XML:
isponygirl

returns true if your slave is a ponygirl/boy

 eg
 actionscript:
if (IsPonyGirl()) {
   // ponygirl
}

xml:
<if isponygirl='true'>
   <!-- ponygirl -->
</if>

Note: This function internally uses the variable
_root.DonePonygirl
         1 = she is a ponygirl
         0 = not yet agreed
You can directly change this in actionscript but generally do not.

PositionTail

actionscript:
function PositionTail(xpos:Number, ypos:Number, rot:Number, wid:Number, hei:Number, cr:Boolean, frame:Number)
XML:
<Tail x='xpos' y='ypos' rotation='rot' width='wid' height='hei' frame='frame'/>

Position a tail behind the slave's image in the main window. The dress image must have a transparent background for the tail to be seen. This one function shows either a Pony tail, a Cat Tail, a Demon Tail or a Puppy Tail. Depending on the current training for your slave the particular type of tail is shown. Except that, the demon tail is shown WITH one of the other tails.

xpos = horizontal position in the main window, 0-480, but you can exceed these limits to have the image partially out of the window
ypos = vertical position in the main window, 0-336, but you can exceed these limits to have the image partially out of the window
rot = rotate the image clockwise in degrees, < 0 for counter-clockwise
wid = width of the image. Will resize the image, will not preserve aspect ratio. Use a negative value to flip the image horizontally.
hei = height of the image. Will resize the image, will not preserve aspect ratio Use a negative value to flip the image vertically.
following parameters are optional
cr = legacy, always use true, not supported by xml. Defaults to true
gframe = the particular version of the tail. Not used for pony tails. Defaults to 1

Note: the debugging menu can create these parameters for both actionscript and xml

PositionGag

actionscript:
function PositionGag(gframe:Number, xpos:Number, ypos:Number, rot:Number, wid:Number, hei:Number, cr:Boolean)
XML:
<BitGag x='xpos' y='ypos' rotation='rot' width='wid' height='hei' frame='frame'/>

Position a gag over the slave's image in the main window.

gframe = the particular version of the gag. Values 1-5 but can vary if more are added to the game. The x,y,wid,hei parameters will vary based on this selection.
xpos = horizontal position in the main window, 0-480, but you can exceed these limits to have the image partially out of the window
ypos = vertical position in the main window, 0-336, but you can exceed these limits to have the image partially out of the window
rot = rotate the image clockwise in degrees, < 0 for counter-clockwise
wid = width of the image. Will resize the image, will not preserve aspect ratio. Use a negative value to flip the image horizontally.
hei = height of the image. Will resize the image, will not preserve aspect ratio Use a negative value to flip the image vertically.
following parameter is optional
cr = legacy, always use true, not supported by xml. Defaults to true

Note: the debugging menu can create these parameters for both actionscript and xml

Slave Girl Interface Functions

These functions can be implemented / overloaded for a slave to control their ponygirl/ponyboy training.

Both of the following function are optional and a generic image is shown if they are not implemented.

ShowEndingPonygirl

actionscript:
function ShowEndingPonygirl() : Boolean
XML:
no specific implementation, the game calls ShowSexActPonygirl()

Show an image from the ending ponygirl. Return true if you show an image, otherwise the game will call ShowSexActPonygirl()

ShowSexActPonygirl


actionscript:
function ShowSexActPonygirl()
XML:
images defined
<Images>
    <Ponygirl>

actual act
<DoPlanning>
    <Acts>
         <Ponygirl>

Show an image and/or text when your slave becomes a ponygirl/ponyboy

Note: for xml training implement in the slaves xml file OUTSIDE of the <Slave> node
<DoPlanning>
    <Acts>
        <Ponygirl>
             put your custom text or code here
       </Ponygirl>
    </Acts>
</DoPlanning>
