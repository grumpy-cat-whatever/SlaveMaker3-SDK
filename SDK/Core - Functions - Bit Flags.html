BitFlags
Slave Maker 3 SDK Updated for Version 3.4

All slaves, places, people, events and your slave maker have a set of bitflags. These are a set of boolean flags, rach of which can have the value of true or false. By default all flags are false unless otherwise noted. There is a theoretically unlimited number of individual boolean flags but in practise using version large numbers will greatly increase the size of save games.

Class

All bitflags in the game are instances of the class BitFlags
Common Scripts/Classes/BitFlags.as
Class BitFlags (simplified, code bodies and private functions and variables omitted)
class BitFlags { 
    public function BitFlags();

    public function Load(lo:Object);
    public function Save(so:Object);
   
    public function CheckBitFlag(flag:Number) : Boolean;
    public function SetBitFlag(flag:Number);
    public function ClearBitFlag(flag:Number);
    public function CheckAndSetBitFlag(flag:Number) : Boolean;  // if the flag is false return true and set the flag to true
   
    public function ShowBitFlags(cnt:Number) : String;
}

Class Methods

CheckBitFlag

actionscript:
function CheckBitFlag(no:Number) : Boolean
XML:
<if flagxxxxxx-no='true'>
where xxxxxx is the name of the event, slave, place or 1,2,SM, Advantages, Skills, InitialItems. See Defined Bitflags

Returns true if the flag is set

 SetBitFlag

actionscript:
function SetBitFlag(no:Number)
XML:
<SetFlag flagxxxxxx-no='true'>
where xxxxxx is the name of the event, slave, place or 1,2,SM etc. See Defined Bitflags

Sets the flag to true. The xml method can set to either true or false.

ClearBitFlag

actionscript:
function ClearBitFlag(no:Number)
XML:
<SetFlag flagxxxxxx-no='false'>
where xxxxxx is the name of the event, slave, place or 1,2,SM. See Defined Bitflags

Sets the flag to false The xml method can set to either true or false.

CheckAndSetBitFlag

actionscript:
function CheckAndSetBitFlag(no:Number) : Boolean
XML:
<SetFlag flagxxxxxx-no=''>

If the selected bitflag is false, the flag is set to true and the function returns true. Otherwise th function returns false.

eg
actionscript:
if (_root.BitFlag2.CheckAndSetBitFlag(5)) {
    // do event, will happen only once as the flag is now set
}

same as

if (_root.BitFlag2.CheckBitFlag(5)) {
    _root.BitFlag2.SetBitFlag(5);
    // do event
}.

General Usage Notes:

BitFlags are a class so to create one for your slave, event they must be created using new
eg
var MyFlags:BitFlags;

In an initialise function
MyFlag = new BitFlags();

To Load or Save a set of flags

cookie.data.MyFlags = new BitFlags();
MyFlags.Save(cookie.data.MyFlags);

To load for a custom event

delete MyFlags;
MyFlags = new BitFlags();
MyFlags.Load(cookie.data.MyFlags);

Defined BitFlags

Slave

Each slave, assistant and minor slave has these sets ot flags defined. These are all member of the class Slave,
eg
var sd:Slave = _root.GetSlaveDetails("Maran");
sd.BitFlag1.SetBitFlag(61);

For the legacy reasons,these flags can be access directly for the the current slave in training, ie without referencing the class instance.
eg
_root.BitFlag1.SetBitFlag(61);
BitFlag1
 core game flags
BitFlag2 custom for slave use
DemonFlags
 state of devil girl training
FairyFlags
 defined for faerie events, but currently unused
ItemsOwned
 defines what items are owned, intended as a replacement for the variables like BitGagOK
ItemsWorn
 defines what items are owned, intended as a replacement for the variables like BitGagWorn

DO NOT use the flags ItemsOwned and ItemsWorn, they are listed here for completeness, but are used internally by the SetItemOwned and IsItemOwned functions. See Core - Items + Dresses.html.
Similarly FairyFlags are reserved, please do not use for a slave's training, or if needed please request a flag.

Slave Maker

Your Slave Maker has these sets of flags, all can be referenced directly or via the SMData object
eg
_root.BitFlagSM.SetBitFlag(0);
or
_root.SMData.BitFlagSM.SetBitFlag(0);
BitFlagSM
 slave maker related, persists for all slave girl trainings
OtherBooks defines specialist books owned (Ladies Guide for instance)
SMInitialItems defines what item related options where purchased during slave maker creation
SMSkills defines skills selected during slave maker creation
SMAdvantages defines the advantages/problems the slave maker ha
SpecialEventFlags set of flags for the slave makers special event

DO NOT use the flag OtherBooks, it is listed here for completeness, but is used internally by the SetItemOwned and IsItemOwned functions. See Core - Items + Dresses.html


Otherwise all places, NPC's and events have a single set of bit flags. The bitflags here do not have a specific instance as their base class inherits from the class BitFlags
eg
Palace.SetBitFlag(10);
<SetFlag flagpalace-10='true'/>

See elsewhere in the SDK for details on the places, NPC's and events for their defined flag values
