Slave's Owners
Slave Maker 3 SDK Updated for version 3.4.02

Each slave has an owner, this is either another person, or your current slave maker. Assistants also can be sort of 'owned'.

Assistants, Minor Slaves and Slaves all handle this concept differently and will be discussed separately.

In all cases the core function SMData.IsSlaveOwned() will return true if the slave is owned, and false if they are not.

Assistants

Assistants are not owned as such, they are free people, commonly fellow Slave Makers. If they work with you long enough they may fall in love with your Slave Maker and confess their love. If you accept their love they will move in and live with you.

In this situation their variable SlaveType (in their instance of the class Slave) will be set to a value of 2, and the function IsSlaveOwned() will return true.

For assistants the member Owner and StartOwner (see Slaves below) are set to null as they are not relevant.

Minor Slaves

The ownership of a minor slave is also controlled by the SlaveType member variable once they have been purchased. Before purchase they have no entry in the Slaves Array (except for one special case). Otherwise the member SlaveType will contain

0
 Owned by yourself
-3
 Sold to another (and IsSlaveOwned() will return false)
-200
 Minor slave that is not yet owned. A very limited case in the game, currently only happens for Arak

For minor slaves the member Owner and StartOwner (see Slaves below) are set to null as they are not relevant.

Slaves

Owner Ids

The slave's owner has a numeric id that the game uses to determine who they are. This value can be the following values
0
 unowned, for Freelancer Slave Makers - default for Freelance Slave Makers
1-999
 one of the standard NPC's, see Core - People.html
1001
 a generic male owner
1002
 a generic female owner
1003
 a generic dickgirl owner
1999
 a generic owner, commissioned by the Slave Maker guild - default for Guild Members
3000
 your Slave Maker

This value is accessed using the GetOwner() method of the PersonOwner class

eg
actionscript:
if (_root.Owner.GetOwner() == 8) {
   // Lord owns slave
}
xml:
<if owner='8'>
   <!-- owner is the lord-->
</if>

Owner & StartOwner

Each slave has member variables defining their Owner. This is an instance of the class PersonOwner which inherits from the class BaseMofule. The class is simplified below with the significant members listed.

Class PersonOwner (simplified)
class PersonOwner extends BaseModule {
       
    var Id:Number;

    public function PersonOwner(slave:Object, cg:Object);
    public function Load(lo:Object);
    public function Save(so:Object)
   
    public function ChangeOwner(id:Number, gender:Number, pname:String) : Number;
   
    function GetOwner() : Number;
   
    function GetOwnerName() : String;
}

For each major slave (see the class Slave) there are the following member variables

    public var Owner:PersonOwner;
    public var StartOwner:PersonOwner;

These variables are initially set to null, and are set to a class instance once the slave starts training. The variable Owner is the current owner of the slave. The variable StartOwner is the slave's owner when training starts. These can be compared to tell if your slave has changed owner.

To access these use the following variables

var sd:Slave = _root.GetSlaveDetails("Ranma");

StartOwner

 startowner
(PersonOwner) As for Owner this is the slave's owner at the start of training. It is set after StartGame() or <StartGame> is called.

Obsolete: SoldSlave

There was in previous versions a variable
_root.SoldSlave
defining who the slave was sold to, default 0. For legacy reasons this variable will still be set and will be the same as the owner id.

DO NOT use this variable anymore.

Methods

The class has the following methods that can be used

ChangeOwner

actionscript:
function ChangeOwner(id:Number, gender:Number, pname:String) : Number
XML:
<ChangeOwner slave='slave' number='id' name='pname' gender='gender'>val</ChangeOwner>

Changes the owner of the slave. See below for the id/number or owner id. Gender uses the standard gender numbers/names as used in other xml statements. The name is the actual name of the owner.

You can only use this for the current slave or a previously trained slave. If you try this for a minor slave or assistant then nothing will happen.
eg
 actionscript:
_root.Owner.ChangeOwner(2050, 2, "Mistress Payne");
or
var sd:Slave = _root.GetSlaveDetails("Ranma");
sd.Owmer.ChangeOwner(2060, 1, "Genma");
xml:
<ChangeOwner name='Genma' gender='1' number='2060'/>
or
<ChangeOwner slave='Tifa' name='Genma' gender='1' number='2060'/>

GetOwner

actionscript:
function GetOwner() : Number
XML:
see owner xml variable above

Returns the owner id for the current owner
eg
actionscript:
if (_root.Owner.GetOwner() == 8) {
   // Lord owns slave
}
xml:
<if owner='8'>
   <!-- owner is the lord-->
</if>

GetOwnerName

actionscript:
function GetOwnerName() : String
XML:
#ownername

Returns the name of the owner. For the general numbers 1000-1999 it will just return "Owner", For yourself (id 3000) it will return "You". Otherwise the name of the owner.

Changing Owners

To change the owner of a slave you can use the ChangeOwner() method of the PersonOwner class or the <ChangeOwner> xml statement, see below.

If you wish to override the owner of the slave at the start of the game, then call these methods in the StartGame() function or <StartGame> node. This will also set StartOwner to the same value.

Selling Owned Slaves

If you sell a slave the SlaveType variable for the slave will be set to -3 to reflect the slave has been sold.

Ending

Ending Number: N/A

The Owner name is shown in the ending screens. If you see the slave to a NPC like the Lord then the normal end process is bypassed and you get some special text reflecting the sale.

Slave Interface Functions

These functions can be implemented/overloaded for a slave to control their cumslut training.

OwnerTest & OwnerTestSpecial

actionscript:
function OwnerTest() : Boolean
function OwnerTestSpecial() : Boolean
XML:
<Events>
     <OwnerTest>
     <OwnerTestSpecial>

Called on days when the slave will be tested (every 7 days of training). Return true to override and replace the standard event.

EventBuyer

assistant/event
