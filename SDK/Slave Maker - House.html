General Concepts

Your Slave Maker has a house they live in, in the current city the game is set in. In theory you can have a different house in different cities. The folliowing does not discuss custom houses, just the actual home you live in. See here for custom homes.

Moving Homes

Between training slaves you can select a new house in your current city for a large fee. You can force this if needed by calling
_root.HouseEvents.MoveHouse();

House ID's

Each house has a unique id that is used to reference it. It is possible to not use the number if you use the function IsHouse() below especially if a 3rd party adds an additional house to the game, as it will be assigned a number.

Map Location

Each house has a location as defined in it's xml. This can vary from city to city and thus there is a separate Housing.xml file for each city.


Basic House Object

Your home is stored in an object in the _root scope in the City data object currentCity

var Home:Housing;

an instance of this class
Common Scripts/Classes/HouseDetails.as
Class HouseDetails (simplified)
class HouseDetails extends Place {
    public var HouseType:Number;            // type of the house. An index into the list of <Houses> in Housing.xml
    public var HouseName:String;            // the name of the house
    public var CityName:String;                // name of the city the house is in
   
    public var Price:Number;                // purchase price of the house

    public var DaysOccupied:Number;            // total days you have lived in the home
   
    // Cost of living
    public var hIncome:Number;                // GP per day this house earns
    public var hUpkeep:Number;                // GP per day required to occupy the home

    // required levels of security and home maintenance/cleaning for the house
    public var BaseHouseWork:Number;
    public var BaseSecurity:Number
    public var ActualHouseWork:Number;
    public var ActualSecurity:Number;
    public var ActualCooking:Number;

    // following are facilities of the house.
   
    // The number represents quality of the facility
    // 0 = not present
    // 1 = standard
    // 2 = food quality
    // 3 = excellent quality
    public var hKitchen:Number;           
    public var hLibrary:Number;
    public var hDungeon:Number;
    public var hWards:Number;
       
    // no quality, just present or not
    public var hStables:Boolean;
    public var hMilkBarn:Boolean;
       
    // The rooms of the house
    public var totRooms:Number;            // total current rooms
    public var maxRooms:Number;            // maximum possible rooms after upgraded/renovations
   
    public var Rooms:Array;                // List of all rooms
   
    // when a background is shown using ShowBath(), ShowBedRoom() etc, these are the selected frames
    // shown for the current house
    public var HomeRoom:Number;
    public var HomeBath:Number;
    public var HomeDungeon:Number;
    public var HomeKitchen:Number;
}

Member variables

HouseType - the type of house
         1 = Castle
         2 = Forest
         3 = Farm
         4 = Mansion
         5 = Apartment
         6 = Temple
         7 = Onsen
         8 = Brothel
         9 = Eerie
         10 = Galley
         11+ for custom or newer houses
hKitchen - the quality of your kitchen,
         1 = basic
         2 = good
         3 = very good
hLibrary - the quality of your library
         0 = none
         1 = good
         2 = very good
hDungeon - the quality of your dungeon
         0 = none
         1 = good
         2 = very good
hWards - are magical wards present
         0 = none
         1 = present
hIncome - how much income does the house generate, in GP per day
hUpkeep - how much money the house requires for maintenance per day

HomeRoom, HomeBath, HomeDungeon, HomeKitchen are the frame in the backgrounds shown when the rooms are used

BaseSecurity - the minimum security needed for the house
BaseHouseWork - the minimum housework needed to maintain the house

ActualSecurity - the allocated security based on slaves orders and your assistant
ActualHouseWork - the allocated housework based on slaves orders and your assistant
ActualCooking - the allocated security based on slaves orders and your assistant

Notes

The house instance is created by the core game when you select a house to live in or load a game. The save data for the house is store in the city save data in
smxcities.sol

The member functions of the Housing class are primarily for internal use, also note that it is derived fron Place base class so all members and variables of that class are available.

A custom house also has an instance of the HouseDetails class.

Cooking needed is the total number of people living in the house divided by 6
Housework needed is BaseHousework + (total number of people / 6)
Security needed is BaseSecurity + (total slaves unsupervsed / 4)
where a slave is unsupervised if not trained or talked to in a given day. Your slave maker counts as 1 point of security and your assistant as 2 points.

Core Functions

These functions are present in the object _root.HouseEvents

ChangeHouse

actionscript:
function ChangeHouse(house)
XML:
<ChangeHouse>house</ChangeHouse>

Change to a new house, you may specify for house either the house id or the name of the house or the folder name of the house images.

 eg
 actionscript:
_root.HouseEvents.ChangeHouse(9);

 xml:
<ChangeHouse>Abandoned Keep</ChangeHouse>

IsHouse

actionscript:
function IsHouse(id) : Boolean
XML:
<if house='id'>

Returns true if the current house is the specified home. You can pass either a numeric id or the name of the house

 eg
 actionscript:
if (_root.currentCity.Home.IsHouse(1)) {
   _root.SetText("The Abandoned Keep is cold and dirty...");
}
if (_root.currentCity.Home.IsHouse("Abandoned Keep")) {
   _root.SetText("The Abandoned Keep is cold and dirty...");
}

 xml:
<if house='1'>
</if>

Custom Houses and Exploring your Home

Slaves, Events, Assistants can add a custom house that can be explored using the "Talk A Walk" planning action.  These custom houses and also your stantdard home can be explored during the 'Talk A Walk' process or other events. These are used by some slaves to represent a cave, a bar, a sorcerors tower, whatever the developer desires.

For more details on this please see Core - House Exploration.


Creating Houses

Basics

The game supports an unlimited list of homes for the game. These are purely handled using xml. The default file is
Languages/English/Common Housing.xml for houses that are common to all cities, ie generic homes that are present in all cities.

Houses that are specific to a particular city are stored in
Languages/English/Cities/cityname/Housing.xml
eg
Languages/English/Cities/Mardukane/Housing.xml

Additional houses can be added by editing this xml or using an external event or by adding an additional xml and configuration xml. For instance see the WeaponsChaoswings.xml mod.
IMPORTANT: In the city veraion of Housing.xml if you wish to update a common house say to alter map positioning or description you must repeat the <Folder> node for the house.
eg
<House id='3'>
    <Name>Farm House</Name>
    <Folder>Images/Cities/Mardukane/Houses/FarmHouse</Folder>
    <MapPosition type='left' x='14.5' y='92.5'/>
</House>

House XML

A house is stored as a child node of <Houses>. Each house is a node
<House> - base node, with an optional id attribute for standard houses with pre-defined events. Do not specify for new houses.

Details of the House

<Name> name of the house
<City> the name of the city this house is in, leave blank for a house common to all cities
<Price> the purchase price for the house, defaults to 10000GP
<Description> long description of the house
<Details> specific effects of the house
<Introduction1> introduction text/images for the house
<Introduction2> optional second page only (no page 3)
<Income> number of GP earned per day
<Upkeep> number of GP per day it costs to live there (default 6GP)
<BaseSecurity> hours of security needed per day (default 1)
<BaseHouseWork> hours of cleaning/maintenance needed per day (default 1)
<TotalRooms> available rooms for use in the house (default 6)
<MaximumRooms> number of rooms in total after house upgrades (default 10)

Areas

<Areas> details facilities in the house Each can have a value of none, standard, good, excellent
Dungeon
Library
Wards
Kitchen
Stables
Barn

eg
<Areas>
    <Dungeon>standard</Dungeon>
</Areas>

Images

<Folder> base folder for loading images
<ShowOutside> called to show the standard outside image for the house
<ShowInside> called to show a standard inside room
<MapPosition> location of the house on the take a walk map.     type means is the label House X  or X House, left means the X is on the left
    x, y positions where the map is 800x600 pixels

Events of the House

<Initialise> called on game load or house purchase
<StartNewSlave> stat/item effects applied to the slave at start of training
<UpdateDateAndItems> effects applied every day. the variable general is the number of days passed, almost always 1
<Events> These will be merged with the standard <Events> node of the game.

eg
<House id='3'>
    <Name>Farm House</Name>
    <Folder>Images/Cities/Mardukane/Houses/FarmHouse</Folder>
    <MapPosition type='left' x='14.5' y='92.5'/>

    <Description>A small farm house on a small parcel of land. Empty for a long time, it seems some odd type of farming was done here. Small, even human-sized stalls. A number of odd items are left.

    You realise this was a ponygirl farm, run as both a working farm and as some sort of brothel.

    Very easy and secure access to the city, with even the occasional noble carriage.</Description>
    <Details>Start with Bondage gear and a bit-gag. You start with knowledge of ponygirls, and all ponygirl actions are easier..&lt;br&gt;-1 Fatigue per day due to easy transport. Starting Temperament and Refinement are lower and your slave becomes more submissive as time passes.</Details>
    <Introduction1>
        <SetText>You move into the odd farm house and you can immediately tell that this place was a brothel, but of an odd nature. What records remain show the girls were kept in the stables in very strict bondage. They were ponygirls, treated like human ponies, bound and made to work, be ridden and pull small carts. They were kept almost naked and sexually used if the customers wanted.
   
        The farm is well fitted out with plenty of rooms and comfortable stalls for ponygirls. There is some left-over gear from the previous occupants, all bondage related. The grounds are large, partly set aside for fields for 'ploughing' and exercise.</SetText>
        <Backgrounds place='farm'/>
    </Introduction1>
   
    <AffectSlave>
        <OwnItem>43</OwnItem>
        <OwnItem>14</OwnItem>
        <ChangeVar temperament='-3'/>
        <ChangeVar refinement='-3'/>
        <ChangeVar difficulty-bondage='-5'/>
        <ChangeVar difficulty-ponygirl='-5'/>
        <SetVar ponygirlaware='1'/>
    </AffectSlave>
   
    <UpdateDateAndItems>
        <SetVar tiredness='tiredness - general'/>
        <SetVar temperament='temperament - general'/>
    </UpdateDateAndItems>
   
    <Areas>
        <Stables>true</Stables>
    </Areas>
   
</House>
