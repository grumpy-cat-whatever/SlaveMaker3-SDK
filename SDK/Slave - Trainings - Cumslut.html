Cumslut Training
Slave Maker 3 SDK Updated version 3.4.02

There are a few variables and functions for your slave's training as a cumslut.

Kasumi
This covers all slaves except Kasumi, her cumslut training is a prerequisite for all cumslut training and is handled internally, without most standard events. While training Kasumi the variables noted below all still apply as do the core functions. The events are handled differently.

Core Variables

Actionscript Variable XML Variable

CumslutLevel

implemented in: _root, class Slave via SlaveSkills
 cumslut
(Number) The level of the slave's training as a cumslut, values
         0-100 = level of training
         2000 = finished training, now a complete cumslut

 Please use ChangeCumslut() or <ChangeCumslut> to alter this skill.

sSlutTrainer

implemented in: _root, classes SlaveMaker and Slave via SlaveMakerSkills 		sluttrainer
(Number) Your Slave Maker's skill level in training sluts. At level 2+ they can train cumsluts.

 Also assistants and slaves have this skill and can be checked via their Slave.

StandardCSText


implemented in: _root
 standardcstext
(Boolean) For a sex act, if true show the standard text for your slave loving cum. Set to false to suppress the standard text. Defaults to true

Events

Start Training

Your slave starts training as a cumslut automatically if IsCumslutTraining() returns true, this happens if your Slave Maker or assistant have Slut Trainer skill at level 2, or your assistant is Cumslut Kasumi.

Cumslut Declaration

You slave will ask to become a cumslut when their CumslutLevel reaches 80 (or 60 for Slave Makers with the Unusual Cum advantage). Several slaves are exempt from this,
  • Brain Slug
  • Arak
Every morning the game checks all slaves (including minor slaves) to see if this is true and if so ONE slave will make their declaration. If more than one slave applies then the later slaves will be delayed to another day.

The game standardly shows the image "Morning Mouthful" via the slave interface function
function ShowMorningMouthfull()
or for xml
<Images><MorningMouthfull>

The declaration will ask a question, triggering the following events.

Question
2500 - Do not become a cumslut
2501 - Become a cumslut

These questions are asked via xml and using the node <CumslutDeclaration>
eg
<Other>
   ...
   <CumslutDeclaration>
        <SetText>#person approaches you nervously and says,</SetText>
        <PersonSpeak person='#person'>Since you started training me I have had been tasting cum more and more and have found that I enjoy the taste. It is lovely and I find myself wanting it often.\r\rI am troubled, this desire is distracting me as sometimes I would prefer to taste and drink cum more than anything?</PersonSpeak>
        <AddText>You feel you should either make #personhimher focus on #personhisher trainings, not just on cum, or you should use this opportunity to have #personhimher change #personhisher life and become a different slave, a Cumslut.</AddText>
        <AskQuestions caption="How do you respond">
            <Question event='2500'>Remind #personhimher of #personhisher training</Question>
            <Question event='2501'>Call #personhimher a cumslut</Question>
        </AskQuestions>
   </CumslutDeclaration>
   ...
</Other>

see the file
Languages/English/Other.xml for the standard version.

Each slave or assistant can implement their own version of this declaration. Implment this inside the slave's <Slave> or <Assistant> node
eg
<Slave>
    .....
   <Other>
      <CumslutDeclaration>
           <SetText>#slave eagerly asks to become a cumslut!!!</SetText>
           <AskQuestions caption="How do you respond">
               <Question event='2500'>Remind #personhimher of #personhisher training</Question>
               <Question event='2501'>Call #personhimher a cumslut</Question>
           </AskQuestions>
      </CumslutDeclaration>
   </Other>
</Slave>

Acceptance and Refusal

If you choose to accept or refuse the declaration the event 2500 or 2501 will be triggered. These then call in a similar way to the above the nodes

<Other>
    <CumslutAgreed>
or
<Other>
    <CumslutRefused>

BUT here the nodes are called as well as the standard handling for these decisions. Standard effects and images are already shown have taken effect. These node allow you to customise the text or apply additional effects.

Again see the file
Languages/English/Other.xml for the standard versions.

Notes

1) The above nodes can be implemented for slaves, assistants and minor slaves to customise these events.

2) There are no actionscript methods provided to alter the events above.

Ending

Ending Number: 27 (special)
Condition:  IsCumslut()

Images are shown using the interface function ShowEndingCumslut, see below. If this function is not implemented or returns false, the game will call ShowMorningMouthfull()

This ending is a 'meta' ending and will always happen when the conditions is true. The game then resets the ending process and selects an actual ending as well as the standard ending.

Core Functions

IsCumslut

 implemented in: _root and Slave class
actionscript:
function IsCumslut() : Boolean
XML:
iscumslut
iscumslut-slavename

returns true if your slave has declared and been accepted as a cumslut.

 eg
 actionscript:
if (IsCumslut()) {
   // cumslut
}

xml:
<if iscumslut='true'>
   <!-- cumslut -->
</if>
<if iscumslut-maran='true'>
   <!-- maran is a cumslut -->
</if>

IsCumslutTraining

 implemented in: _root and Slave class
actionscript:
function IsCumslutTraining() : Boolean
XML:
iscumsluttraining

returns true if your slave can train as a cumslut

ChangeCumslut

 implemented in: _root and Slave class
actionscript:
function ChangeCumslut(val:Number)
XML:
<ChangeCumslut>val</ChangeCumslut>

Change cumslut training by val points. Has no effect if cumslut training has not started.

 eg
 actionscript:
_root.ChangeCumslut(5);

 xml:
<ChangeCumslut>5</ChangeCumslut>
<ChangeCumslut slave='Tifa'>5</ChangeCumslut>


Slave Interface Functions

These functions can be implemented/overloaded for a slave to control their cumslut training.

ShowEndingCumslut

actionscript:
function ShowEndingCumslut() L Boolean
XML:
no specific implementation, will always use the Morning Mouthfull image

Called when the ending for a cumslut slave is shown. Generally show here the image of the slave enjoying and covered in cum. Return true if you show an image, false to use a generic image.

Note this function is optional, if you do not implement the game will show a generic image.

eg
 actionscript:
function ShowEndingCumslut() : Boolean
{
   _root.Backgrounds.ShowBedRoom();
   _root.ShowMovie(EndingCumslut, 1, 3);
   return true;
}
