General Concepts

Rules affects a slaves training and generally apply a stat change on a daily basis.

Rule Numbers

All rules have an associated number that is used in the functions below.. See the column ID for the variabled below for that identifier

Core Variables

All the following variables are implemented in a class TrainingRules with an instance
Rules
in global scope. For legacy support all variables are copied to global scope
ie
_root.Rules.RulesTalk
can also be referred to as
_root.RulesTalk
In actionscript please use the ChangeRuleTo() function to alter the setting of a rule

Core Functions


ChangeRuleTo

actionscript:
function ChangeRuleTo(rule:Number, val:Number)
XML:
no equivallent, just change the listed variable above

Use this to change a rule,

eg
_root.Rules.ChangeRuleTo(1, 1);
to set the rule to allow the slave to pray

Breaking Rules

There are events that can happen and in them your slave will break one of these rules. Notably the fuck, talk rules. When this happens the game sets a variable

Actionscript Variable XML Variable

BadGirl

implemented in: _root 		badgirl
(Number) Has your slave broken a rule
         0   = No
         1   = Yes

If this variable is set then the obedience effects of the sex act Spanking is increased.

Note:
this is a very minimal mechanic at this time.

Slave Interface Functions

These functions so you can affect the turning on or off of a rule. It only affects when you use the Rules screen, it has no effect if you programaticaly change the rules variables.

These are optional

SetRule

actionscript:
function SetRule(rule:Number) : Boolean
XML:
<Events>
    <SetRule>

Called when you turn on a rule. The parameter is the number refered to in the variable definitiions (eg 1 for RulesTalk).

Return false to prevent the rule being set. Return true and the rule is set as normal. If you want to trigger a complex event then terminate the event with event 9708 to return you back to the rules screen. Do not bother to do so to just disable setting a rule.

XML:
The variable general is set to the rule number. If you execute any non-conditional statement then this will be treated as retuning true. Otherwise false
eg prevent Rule Pray being set
<Events>
      .....
      <SetRule>
          <if general='2''>
                <SetText>#slave says #slaveheshe has no interest in praying!</SetText>
          </if>
      </SetRule>
      .....
</Events>

ClearRule

actionscript:
function ClearRule(rule:Number) : Boolean
XML:
<Events>
    <ClearRule>

Called when you turn off a rule. The parameter is the number referred to in the variable definitions (eg 1 for RulesTalk).

Return false to prevent the rule being set. Return true and the rule is set as normal. If you want to trigger a complex event then terminate the event with event 9708 to return you back to the rules screen. Do not bother to do so to just disable setting a rule.

XML:
The variable general is set to the rule number. If you execute any non-conditional statement then this will be treated as retuning true. Otherwise false
eg prevent rule Talk being turned off
<Events>
      .....
      <ClearRule>
          <if general='1' obedience='90-'>
                <SetText>#slave irefuses to shut up!</SetText>
          </if>
      </ClearRule>
      .....
</Events>
