Love Confession & Dating
Slave Maker 3 SDK Updated for version 3.4.02

Slaves can fall in love with your Slave Maker and potentially with other NPCs (not implemented). Some NPC's can fall in love with your slave and some with your Slave Maker.

Your slave can also start dating other people from a random person to an older lover who pays for their company.

Basic Concepts

Love for your Slave Maker

Slaves have a hidden stat, VarLovePoints/love that reflects their love for your Slave Maker. This increases via certain acts, notably Kiss, and "Talk To Slaves" and some sex acts.

Once this value is 60 or higher there is a chance that during the morning events the slave will confess their love for you. You will have the choice to accept, refuse, or not choose their love. Accepting love is considered inappropriate for a Slave Maker and your score for thraining the slave will be penalised, but you will get the option to retain the slave via the 'Love' ending.

After the first confession your Slave Maker will gain the skill "Leadership" level 1. This skill also affects how many slaves you can be in a relationship with at a time. If your skill is not high enough then the slave will not confess their love in the first place but an event will happen when they back off i.

Assistant for your Slave Maker

This process also applies to assistants and in you accept their love they will move into your household and offer their services as an assistant for free and be available for general tranings as a participant. They will also allow you to alter their contraceptive status. Once they beome this 'Love Assistant' their variable SlaveType changes to a value of 2.

NPC Loving your Slave (aka Noble Love)

A slave can have a selected NPC become fasinated with them, and fall in love. This is controlled by the variables NobleLoveType and NobleLove. The game tracks the encounters and updates the variable NobleLove and whenever the slave Visits or meets the NPC the interface function NobleLove() is called. The game implements a Yes/No event 13 for the eventual case where the NPC confesses their love, but this is never explicitly triggered, it is up to the slave developer to choose to do this.

Nothing else is done by default by the game, it is up to the slave developer to handle this as desired.See the source code for Akane for an example of this.

Dating

If you do not meet the sexual desires of your slave then they can start dating, assuming you allow them to Go Out in Rules and they are not wearing a Gag.

The criteria are
EventBoyfriend != -1
VarNymphomania > 10
and either
Slutiness > 8
RulesGoOut == 1 and BitGagWorn == 0

Older Lover

At the sleazy bar your slave can meet a person who will pay them for their company, to become their mistress for pay. This is an older man or woman.

Variables

Actionscript Variable XML Variable

VarLovePoints

implemented in: _root, class Slave
 love
initialise in <Slave>: <Lover>
(Number) This represents the slave's love for your Slave Maker. Range 0 to 100. Please only change using the Points() function or <Points> xml statement.
Default 0

xml:
You can initialise this for an xml slave, in the <Slave> node to indicate the slave is already a bit in love with you.
eg
<Slave>
    <Name>Ifurita</Name>
    ....
    <Charisma>10</Charisma>
    <Love>10</Love>
    ....
</Slave>

LoveAccepted

implemented in: _root, class Slave
 loveaccepted
initialise in <Slave>: <LoveAccepted>
(Number)  Have you accepted this slaves love

        -1 = no and they will never confess love
          0 = no, they have not confessed yet
          1 = yes you accepted
          2 = you refused their confession
        10 = yes and you got the "Love" ending to buy the slave back
Default 0

OldLover

implemented in: _root, class Slave
 oldlover
initialise in <Slave>: <OldLover>
(Number) The type of older lover the slave has taken
        -1 = will never take an older lover
         0 = does not have an older lover
         1 = male older lover
         2 = female older lover
Default 0

xml:
You can initialise this for an xml slave, usually to set it to a value of -1 in the <Slave> node
eg
<Slave>
    <Name>Ifurita</Name>
    ....
    <OldLover>-1</OldLover>
    ....
</Slave>

EventBoyfriend

mplemented in: _root
 dating
isdating (READ ONLY)
initialise in <Slave>: <Dating>
(Number) is the slave dating a person. Set to -1 to prevent dating.
Default 0

xml:
You can initialise this for an xml slave, usually to set it to a value of -1 in the <Slave> node

NobleLoveType

mplemented in: _root
 noblelover
initialise in <Slave>: <NobleLove>
(Number) The id of the NPC that can fall in love with your slave. See Core - People.html for details on the values. Set to 0 for no noble lover.
Default 0 (no NPC)

xml:
You can initialise this for an xml slave, usually to set the NPC involved.

NobleLove

mplemented in: _root
 noblelove
(Number) The progress of the love of the NPC. A simple count of the number of encounters with them,
Default 0, can only be changed or initialised by setting the variable
eg
_root.NobleLove += 2;
<ChangeVar noblelove='2'/>

There are only limited cases where you would need to change this variable.

Event Numbers


yes/no events
3        - love confession
13       - sold to NPC ('noble love')
20       - buy her back at end of game

Slave Interface Functions - Dating

These functions can be implemented/overloaded for a slave to control their dating.

ShowDating

actionscript:
function ShowDating()
XML Event:
<Daily>
   <Dating>
XML Images:
<Dating>

Show an image of her dating a lover, or just of her being seductive.

For XML the image is displayed based on the node <Dating>
eg for a single, normal image
<Images>
   <Dating>
       <Normal>1</Normal>
   </Dating>
</Images>

The images are named according to the pattern
Dating x.extension
eg
Dating 1.jpg
Dating 2.png

actionscript:
function OldLoverDating()
XML:
<Daily>
   <DatingOldLover>

Show an image of her dating a lover, or just of her being seductive. Defailt text and stat effects have been applied at this stage.

actionscript:
no implementation

Displays text when the slave breaks up with their older lover. Only an xml override is provided.

Override to alter the text/graphics when your slave first meets their older lover


Slave Interface Functions - Love

These functions can be implemented/overloaded for a slave to control their love confessions.


ShowLoveConfession

actionscript:
function ShowLoveConfession()
XML Event:
<Other><LoveConfession>
<Slave><Other><LoveConfession>
for assistants only
<Assistant><Other><LoveConfession>
XML Images:
<Images><LoveConfession>
naming: Love Confession n,jpg

Called when the slave confesses love. No default image is shown for actionscript based slaves. For xml based slaves a generic image is show if no image is shown.

Standard text can be disabled by setting _root.AppendActText = false or in xml appendacttext='false'

For XML slaves the event will preferentially use <Slave> version. Similarly assistants will preferentially use the <Assistant> version. If neither are present the blogal <Other> version is used. Note: the xml event is always called for BOTH xml and actionscript slaves/

At the time the xml event/and the actionscript function is called standard text has been shown and you can replace it using SetText as needed. After they are called the game displays a list of questions <Other><LoveConfessionResponses> giving the standard accept, refuse, unsure. If you have shown a question in the xml event then the standard questions will be appended. Also note <Other><LoveConfessionResponses> can also be present in <Slave> and <Assistant> allowing you to precent questions or change or remove options.

Slave Interface Functions - Noble Love

NobleLove

actionscript:
function NobleLove(alowoffer:Boolean)
XML:
<Other><NobleLove>
variable general is 1 if allowoffer is true

Called when the slave encounters a noble who is interested in them. Generally have an event where they express interest, If allowoffer is true then the noble can offer to purchase the slave.

Core Functions - Noble Love


NobleLoveEvent

actionscript:
function NobleLoveEvent(noble:Number, alowoffer:Boolean)
XML:
<NobleLoveEvent allowoffer='true'>Noble name or number</NobleLoveEvent>

Call this in an event where the slave meets a notable NPC, pass the unique id for the noble, or for the XML version you can pass their name.

If the slave is one the noble is interested in bases on NobleLoveType then the game will call the interface NobleLove()
