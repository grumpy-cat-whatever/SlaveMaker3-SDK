General Concepts

Text Areas

The game displays text in two general places, called text areas
  1. General Text field - this is the central text area in the lower center part of the screen. It is not scrollable and can diaplay abut 18 lines of text.
  2. LargerText field - this is the large area that can replace the statistics boxes, it is scrollable and unlimited in length.
Any text that overflows the General Text Field (ie exceeds 18 lines long) will automatically make the Larger Text field visible and then show the extra text in the Larger Text field.

One point of note is you can sometimes get an odd display of text if you show a single line in either field. The text appears a little cropped at the bottom. So far no good solution has been found for this issue, so if you see this with text you output, please just append some blank lines on the end.

Blank Lines and Escaped Characters

Generally try to space out any text to make it more readable. Using the general text functions this is done using \r
ie
_root.SetText("this is line 1\r\rthis is line 2 with a blank lines before it");

Also remember escaped values, this means to use special character combinations for special characters
\" for a "
\r start new line
\r\r for a blank line

HTML

All text shown by the game is html encoded, this means you can use html tags to format the text, for underlining, boldface, colour etc
ie
_root.AddText("The creature <b>attacks</b> and <font color='red'>bites</font> #slave painfully.\r\rYou run to help...");

In xml this can also be done

     <AddText>The creature <b>attacks</b> and <font color='red'>bites</font> #slave painfully.

     You run to help</AddText>

There are some predefined css styles defined by the game
<ps> - colours the text the standard colour for speech in the game
eg
<ps>Boo</ps>

<redtext> <bluetext> <greetext> - colours the text red/blue/green

<ld2> <ld3> <ld4> - alters line spacing

XML

As you see above xml works similarly to actionscript with a few variations
  • you do not need \r just add a blank line to the text
  • all whitespace (leading spaces and tabs are ignored before text when you use a blank line So in the example above the spaces before "You run to help" are ignored. This is to allow the xml files to be indented for better readability.
  • The game autmotically adds a blank line to the end of most event text, so the note about odd truncations is not relevant to xml.
  • only some common html tags are well supported by xml as used in the example above. Only <b> <font> <u> <p> <i> and the syles <bluetext> <redtext> <greentext> <ld3> <ld4> <ld2> <ps>. You can use any or all tags, but you will have to use a syntax like &lt;table&gt; etc, ie replace all <> with &lt; and &gt;

Functions

General

These are general functions to display text in the game, all are implemented in a class GameLanguage, with a runtime instance Language.
eg
_root.Language.AddText("Hello");

For legacy reasons all these functions are also implenented in root scope
eg
function AddText(s:String) { Language.AddText(s); }
to allow
_root.AddText("Hello");

Additionally most of these functions are implemented similarly the SlaveModule class so any events or slaves that extend that class can use
AddText("Hello");


AddText

 implemented in: _root and SlaveModule class
actionscript:
function AddText(nstring:String)
XML:
<AddText>string</AddText>

Adds text to the games text areas, First to the General Text area and will overflow into the Larger Text area. This function is also implemented in the SlaveModule class via the base class BaseModule. This means in any slave or event derived from the SlaveModule class you do not require the _root. prefix.

 eg
 actionscript:
 _root.AddText("<b>I'm bold</b>");
_root.AddText(" and <i>sexy</i>");

(in a class derived from SlaveModule)
SetText("You are walking down the street ");
if (Talent == 21) AddText("and you are aware of a tentacle monster nearby.");
else AddText("and you smell an odd smell.");
AddText("\r\rThe thing attacks...");

 xml:
<AddText><b>I'm bold</b></AddText>
<AddText> and <i>sexy</i></AddText>

SetText

 implemented in: _root and SlaveModule class
actionscript:
function SetText(nstring:String)
XML:
<SetText>string</SetText>

Works identically to AddText but first it erases all text in all text areas, so it starts with a clean text field. Useful for cases when you override an existing event.

AddGeneralText

 implemented in: _root and SlaveModule class
actionscript:
function AddGeneralText(nstring:String)

no xml implementation

Adds text ONLY to the General Text area. The text does not overflow into the Larger Text area. Mainly used for hints and shopping type text.

SetGeneralText

 implemented in: _root and SlaveModule class

 implemented in: _root and SlaveModule class

no implementation

AddTextToStartGeneral

BlankLine

 implemented in: _root and SlaveModule class
actionscript:
function BlankLine()
XML:
<BlankLine/>

Adds a blank line to the end of the currently shown text. Essentially a shortcut for
_root.AddText("\r\r");

Speech

These are function to display text as if someone is speaking, generally in the format like
Ranma:
"Hello there"
The function also support modifying the text if the speaker is a catgirl, ponygirl, puppygirl, or is wearing a gag. ie
Shampoo briefy removes Ranma's gag
Ranma:
"Hello there, Neigh!"

These modifications are automatically dome for the Slave and Servant versions of the functions, and by the Slave class versions.
eg
_root.SlaveSpeak("Hello there");
_root.AssistantData.SlaveSpeak("Hello there");
_root.ServantSpeak("Hello there");
will all modify the text.

PersonSpeak

 implemented in: _root and SlaveModule class
actionscript:
function PersonSpeak(person:Object, say:String, newl:Boolean)
XML:
<PersonSpeak person='person'>say</PersonSpeak>
you can interchangably use slave='person'

The person will speak the text specified. The person parameter can be any of the following
  • a general name "Trader" "Party Goer" etc
  • a standard NPC's person name "Lady Farun" "Miss. N" etc
  • a person number, see Core - People.html or the xml file Languages/English/Base.xml and the People nodes
Actionscript and XML have some differences

actionscript:
 By default all text is cleared when this function is called (ie calls SetText()) but you can make it append text by using the new line parameter
 eg
 _root.PersonSpeak("Lady Farun", "Hi!", true);

 xml:
By default text is appended, if you wish to clear all text first then add a line <SetText/> before the call.
When <PersonSpeak> is called the game adds a blank line before and after the displayed text.
eg
<PersonSpeak person='Lady Farun">Hi</PersonSpeak>
In actionscript this would be
_root.BlankLine();
_root.PersonSpeak("Lady Farun", "Hi", true);
_root.BlankLine();

There are also several standard person values for the xml version
  • assistant - your current servant
  • slave - your current slave
  • slavea. slaveb - for twins these represent one of the pair picked at random. For non-twins then just the slave
  • assistanta, assistantb - for assistants who are twin, one of them picked at random. For non-twins, then just the assistant
Lastly there are attributes to reproduce the PersonSpeakStart and PersonSpeakEnd functions below
eg
<PersonSpeak person="Lady Farun" start='true'>Hi there </PersonSpeak>
<if gender='male'>
   <PersonSpeak end='true'>handsome.</PersonSpeak>
<else>if</else>
   <PersonSpeak end='true'>beautiful.</PersonSpeak>
</if>

PersonSpeakStart

 implemented in: _root and SlaveModule class
actionscript:
function PersonSpeakStart(person:Object, say:String, newl:Boolean)
XML:
<PersonSpeak person='person' start='true'>say</PersonSpeak>
you can interchangably use slave='person'

Works identically to PersonSpeak but it does not close the speech, by adding the training " or add a line feed. This allows you to create variations of the text depending on gender or other issues.

eg
_root.PersonSpeakStart("Lady Farun", "Hi there ");
if (_root.Gender == 2) _root.AddText("beautiful.");
else _root.AddText("handsome.");

will show
Lady Farun:
"Hi there handsome."

As noted above XML works a bit differently, see the discussion on <PersonSpeak>,

PersonSpeakEnd

 implemented in: _root and SlaveModule class
actionscript:
function PersonSpeakEnd(say:String)
XML:
<PersonSpeak person='person' end='true'>say</PersonSpeak>
you can interchangably use slave='person'

This appends the string to the currently displayed text, and then closes off the speech by adding " and a line feed.

 As noted above XML works a bit differently, see the discussion on <PersonSpeak>,

Slave Class Implementation

SlaveSpeak,SlaveSpeakStart, SlaveSpeakEnd,

ServantSpeak, ServantSpeakStart, ServantSpeakEnd

 implemented in: _root and Slave class
actionscript:
function SlaveSpeak(person:Object, say:String, newl:Boolean)
XML:
just use <PersonSpeak>

as above but specific for your slave/servant. The person parameter is omitted as it is implied.

For Twins there are also Slave1Speak, Slave2Speak, Slave1SpeakStart, Slave2SpeakStart

AddSpeechSuffix

 implemented in: _root and Slave class
actionscript:
function AddSpeechSuffix(chance:Number,stringsuffix:String)
XML Slave Initialisation:
<Speech>
    <Suffix chance='number'>string</Suffix>
</Speech>
XML function:
<AddSpeechSuffix slave='name' chance='number'>string</AddSpeechSuffix>

This will append a phrase or word to the end of any speech the slave give using the SlaveSpeak type functions. It will happen a certain percentage of the time, default is every time. For example a certain character in an anime always add the word 'Desu!' to the end of every time they speak.

For actionscript this can be changed at any time, or just in the StartGame() function, as this setting is remembered and saved for the slave.
eg for the current slave
_root.AddSpeechSuffix(100, "Desu!");
or for a specific slave
var sd:Slave = _root.GetSlaveDetails("Shampoo");
sd.AddSpeechSuffix(50, "Nihao!");
In XML this can be set at anytime using <AddSpeechSuffix> or can be initialised by adding to the slaves xml, in the <Slave> node
eg note ... just represent other nodes
<Slave>
     ....
   <Speech>
       <Suffix chance='100'>Desu!</Suffix>
    </Speech>
    .....
</Slave>

There are a number of generic manipulation functions for altering case, plurality, but these are not needed by a slave girl developer as you know the gender and plurality of your slave.

The following function are only implement for actionscript and are only available via _root scope.

 Prototype
 Description

AddText(SlaveName + NonPlural(" refuse") + " explaining that " + SlaveHeSheIs + " not of their faith. The person smiles and rejoins the others to make their offerings.");

Text Macros

In all the above functions (and most other places text is used) a series of macros can be used

eg
_root.SetText("#slave can compete in a contest again.");


#slave   - will be replaced by the slaves name,
                  eg Ranma
#assistant       - will be replaced by your servants name,
                  eg Shampoo
#slaveheshe - he/she/they based on gender of your slave
#slavehimher- him/her/them based on gender of your slave
#slavehimher2 - for translation cases where there are tense/plural differences
#slavehimher3
#slavehimher4
#slavehisher     - his/her/them based on gender of your slave
#slavehisher2 - for translation cases where there are tense/plural differences
#slavehisher3
#slavehisher4
#slaveA          - SlaveA, usually this is the slave's name. For twins it is one of the slaves picked at random
#slaveB          - SlaveB, the other twin than SlaveA (or SlaveName for non-twins)
#assistant       - your assistants name
#assistantA      - ServantA, as above but for your assistant
#assistantB      - ServantB, ditto
#assistantheshe          - he/she/they based on gender of your assistant
#assistanthimher         - him/her/them based on gender of your assistant
#assistanthimher2 - for translation cases where there are tense/plural differences
#assistanthimher3
#assistanthimher4
#assistanthisher         - his/her/them based on gender of your assistant
#assistanthisher2        - for translation cases where there are tense/plural differences
#assistanthisher3
#assistanthisher4
#super   - the name of the person supervising, either 'you' or your assistants name
#supername       - the name of the person supervising, either your slave makers name or your assistants name

#master          - Master or Mistress, depending on your Slave Makers gender
#slavemakername          - your Slave Makers name
#superhisher     - his/her/them based on gender of the person currently supervising your slave
#superheshe      - he/she/they based on gender of the person currently supervising your slave
#supername       - the name of the person currently supervising your slave, always the persons explicit name
#super   - similar to #supername but show 'you' for your slave maker
#assistantpronoun - the pronoun your asistant uses, default I
#assistant       - your assistants name


First Letter in Uppercase
The macros for he/she/his/her also support displaying He
eg
He goes for a walk
To do this just use #Slaveheshe
eg
#Slaveheshe goes for a walk
