General Concepts

The following documents the functions you can use to access the XML functions for the game. These can be used to execute xml statemes for a nominated xml node

All functions are in a class instance
_root.XMLData

Node References

All functions take a parameter of the node and or subnode. This is the name of the xml node to process

eg
<Events>
      <MyEvent1>
          <!-- various statements -->
          <SetText>testing, testing</SetText>
     </MyEvent1>
</Events>

To reference this
_root.XMLData.XMLGeneral("Events/MyEvent1");

The game uses a path like format of parent node, child node, child node of child node etc. The root node <Language> must be omitted.

Node Searching

When you refer to a node the game searches for the xml node if you do not specify an explicit path as discussed above. The game searches

1. If the node referenced is a child node of the parent node from the location the xml is called. This is not done for the Actionscript functions but is always done for xml statements like <DoXML>

2. Seaches in <Events> node

When using <DoXMLAct> the game also tries

3. Planning/DoPlanning

4. Planning/Acts/DoPlanning

Functions


DoXMLAct

 implemented in: GameXML class instance XMLData
actionscript:
function DoXMLAct(act:String) : Boolean
XML:
<DoXMLAct>act</DoXMLAct>
and/or
<if doxmlact='act'>

Perform the XML for a training the xml is seached for in <Planning><Acts><DoPlanning>. The function returns true if any xml statements are processed excluding any conditionals like <if>
ie
<Library>
     <if elapsed='11'>
           <AddText>do something</AddText>
     </if>
</Library>
So if training is before day 11 then DoXMLAct("Library") will return false as the if statement is false
eg
 actionscript:
 _root.DoXMLAct("Library");
xml:
<DoXMLAct>Library</DoXMLAct>

XMLEvent

 implemented in: GameXML class instance XMLData
actionscript:
function XMLEvent(node:String, sett:Boolean) : Boolean
XML:
<DoXMLEvent>event</DoXMLEvent>
and/or
<if doxmlevent='event'>

Perform an XML event in the <Events> node . As for DoXMLAct() it returns true if any non-conditional statements are executed.
The sett parameter means that if any text is displayed in the event then any existing text will first be cleared (ie use SetText()). Defaults to false. No equivallent for <DoXMLEvent> implementation
eg
 actionscript:
 _root.DoXMLEvent("MyEvent1", true);
xml:
<DoXMLEvent>MyEvent1</DoXMLEvent>

XMLGeneral

 implemented in: GameXML class instance XMLData
actionscript:
function XMLGeneral(node:String, sett:Boolean) : Boolean
XML:
no implementation

Perform an XML node, node where it will search for the tag
1. in <Events>
2. starting at the root, ie the children of <Language>
The sett parameter means that if any text is displayed in the event then any existing text will first be cleared (ie use SetText()). Defaults to false.
eg
 actionscript:
 _root.DoXMLGeneral("Other/LoveConfession", true);

PickAndDoXMLEvent

 implemented in: GameXML class instance XMLData
actionscript:
function PickAndDoXMLEvent(node:String, sett:Boolean) : Boolean
XML:
<PickAndDoXMLEvent>event</PickAndDoXMLEvent>
and/or
<if pickanddoxmlevent='event'>

Looks through an XML Node like
<Events><Daily>
That has a list of nodes with conditionals. It will check all child nodes until one of the nodes conditionals evaluate to true. It will then perform the xml statements for the node
